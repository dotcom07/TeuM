# 틈새움 백업 및 이후 동기화 원칙

## 1. 1.0.5의 결정

틈새움은 **계정 없이 바로 쓰는 앱**을 유지한다. 서버·계정·자동 동기화를 흉내 내지 않는다.

- 기본: 모든 설정, 틈 기록, 픽셀 데스크는 이 기기에만 저장한다.
- 기기 이동: 사용자가 `백업 파일 만들기`로 파일을 저장·공유하고, 새 기기에서 `백업 파일 불러오기`로 직접 복원한다.
- 자동 동기화: 서버와 계정이 실제로 준비된 뒤에만 선택 기능으로 제공한다.

이 방식은 새 기기 이동에 한 단계가 더 필요하지만, 로그인 강제·서버 비용·개인정보 처리 변경 없이 기록과 아이템을 보존한다.

## 2. 백업 파일 계약

파일 확장자는 `.json`이며 권장 이름은 `teum-backup-YYYY-MM-DD.json`이다.

```text
format: "teum-backup"
version: 1
createdAt: ISO 8601 시각
payload:
  persisted: 온보딩·알림 설정·현재 리듬
  records: 틈 기록 (최대 4,000개)
  desk: 누적 챙김·픽셀 데스크 배치
  languageMode: auto | ko | en
```

복원 전에는 형식·버전·필수 값·기록 상한을 검증한다. 사용자가 확인하면 현재 기기의 설정, 기록, 데스크 배치를 파일 내용으로 교체한다. 잘못된 파일은 적용하지 않는다.

## 3. 이후 선택형 계정 동기화

서버가 준비된 후에도 위 `payload`를 계정별 동기화 문서의 본문으로 유지한다. 기존 로컬 사용자에게는 다음 순서를 지킨다.

1. 설정의 `백업 및 기기 간 이동` 안에서만 `동기화 연결`을 제안한다.
2. 첫 연결 시 `이 기기의 기록을 백업하기`와 `클라우드 기록으로 바꾸기`를 명시적으로 고르게 한다. 자동 병합하지 않는다.
3. 로그인하지 않은 사용자의 로컬 사용 흐름·알림·기록은 바꾸지 않는다.
4. 동기화 문서에는 `version`, `updatedAt`, 이후 필요한 `deviceId`를 추가해 마이그레이션과 충돌 처리를 한다.

계정 생성 기능을 출시하는 버전에는 앱 안의 계정·연결 데이터 삭제, 앱 밖에서도 접근 가능한 삭제 요청 URL, 개인정보처리방침과 Play Console 데이터 안전성 갱신을 함께 제공한다.

## 4. 1.0.5 배포 운영

- 코드와 버전은 지금 `1.0.5` / Android `versionCode 6`으로 고정한다.
- 현재 브랜치에서 검증된 커밋을 원격에 푸시한다.
- Play Console에는 지금 업로드하지 않는다. 3일 뒤 동일 커밋으로 만든 AAB만 내부·비공개 테스트 트랙에 올린다.
- 업로드 뒤 코드가 바뀌면 같은 `versionCode 6`을 재사용하지 않는다. 수정은 `1.0.6` / `versionCode 7`로 분리한다.

## 5. 개인정보 경계

백업 파일은 사용자가 명시적으로 만들고 직접 보관·전달한다. 앱은 파일을 개발자 서버로 전송하지 않는다. 파일을 메일·클라우드·메신저에 올리는 경우의 보관·공유 정책은 사용자가 선택한 서비스에 따른다.
