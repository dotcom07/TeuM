# 틈새움 TeuM 서비스 기획 및 화면 설계서

> 버전: MVP 1.2
> 범위: PC 앱, Android 앱  
> 원칙: 계정·서버·외부 전송은 사용하지 않는다. Android의 선택형 기록 모드에서만 사용자가 직접 남긴 O/X 결과를 기기에 저장한다.

## 1. 서비스 정의

### 이름

- 한글명: **틈새움**
- 영문명: **TeuM**
- 앱 아이콘 표기: **TeuM**
- 스토어 표기: **틈새움 TeuM**
- 영문 의미: *One minute for a healthier workday.* — 건강을 중심에 둔 표현을 사용하고, "flow" 같은 생산성 은유는 브랜드 문구에 쓰지 않는다.

### 한 줄 소개

**매시간 물 한 모금과 가벼운 스트레칭을 챙기게 해 주는 직장인 건강 알람.**

### 제품 포지셔닝

틈새움은 **건강 목적의 반복 알람 앱**이다. 사용자는 “스트레칭과 물 마시기를 알려 주는 직장인 건강 앱”으로 이해하고, Android와 Google Play에는 “사용자가 직접 일정과 간격을 설정하는 반복 알람 앱”으로 동작한다.

| 층위 | 정의 | 대표 표현 |
|---|---|---|
| 사용자가 느끼는 가치 | 오래 앉아 일하는 중 물과 스트레칭을 놓치지 않게 돕는다. | `일하는 나를 위한 1분` |
| 제품 카테고리 | 직장인 건강 습관을 위한 반복 알람 | `업무 건강 알람` |
| 시스템 핵심 기능 | 사용자가 정한 시간·요일·간격에 기기에서 정확히 실행되는 알람 | `다음 건강 알람 오전 10:00` |

용어는 역할에 따라 구분한다.

- 설정·권한·예약·다음 시각에는 `건강 알람`을 사용한다.
- 알람이 울린 뒤 사용자가 경험하는 60초에는 `1분의 틈`을 사용한다.
- Android 시스템 권한의 공식 명칭을 설명할 때만 `알림 권한`, `전체 화면 알림 권한`을 사용한다.
- `웰니스 리마인더`, `생산성 타이머`, `휴식 도우미`처럼 알람의 핵심성이 흐려지는 표현은 대표 문구로 사용하지 않는다.

### 해결하려는 문제

직장인은 장시간 앉아 일하면서 물 마시기와 짧은 움직임을 자주 놓친다. 그러나 알림이 잦거나, 강제적이거나, 업무 중 화면을 크게 가리면 오히려 집중을 해친다.

틈새움은 건강을 감시하거나 사용자를 평가하지 않는다. 사용자가 직접 켠 반복 알람으로 **짧고 부담 없는 1분의 틈**을 제안한다.

### MVP의 단일 루틴

알림을 받으면 사용자는 다음을 한다.

1. 물을 몇 모금 마신다.
2. 자리에서 일어나 가볍게 스트레칭한다.
3. 1분 뒤 업무로 돌아간다.

앱은 각 동작을 영상·그림·카운트로 강제하지 않는다. 화면에는 짧은 문장으로만 안내한다.

> 물 한 모금 마시고, 자리에서 일어나 가볍게 스트레칭해 보세요.

## 2. 목표와 비목표

### 목표

- 업무 시간에 1시간 간격으로 1분 휴식의 계기를 만든다.
- 사용자가 다음 알람 시각을 예측하고 언제든 끄거나 미룰 수 있게 한다.
- 알림을 최소한의 방해로 전달한다.
- 사용자가 쉽게 미루고, 쉽게 다시 시작할 수 있게 한다.
- 기록이 귀찮은 사람은 추가 입력 없이 쓰고, 기록을 좋아하는 사람은 달력에서 성취를 확인할 수 있게 한다.
- PC와 Android에서 같은 루틴을 서로 다른 사용 맥락에 맞게 제공한다.

### 비목표

- 물의 양(mL), 걸음 수, 체중, 심박수, 자세, 건강 상태를 측정하거나 추정하지 않는다.
- 기록을 기본값으로 강요하지 않는다. 연속 기록, 랭킹, 경쟁, 목표 미달 경고를 제공하지 않는다.
- 카메라·마이크·위치·연락처·접근성 권한을 사용하지 않는다.
- 의학적 진단, 치료, 건강 효과를 약속하지 않는다.
- 스트레칭 자세를 평가하거나 정확한 동작 수행을 요구하지 않는다.

## 3. HCI 핵심 원칙

| 원칙 | 제품 적용 |
|---|---|
| 방해 최소화 | 한 시간에 알림 하나만 보낸다. Android 전체 화면 알림도 한 회차에 한 번만 열고 반복 재촉하지 않는다. |
| 사용자의 주도권 | Android는 `5분 뒤에 다시`, `네, 챙겼어요`, `이번엔 넘길게요`를 항상 제공한다. |
| 작은 결정 | 알림에서 선택지는 최대 3개로 제한한다. |
| 낮은 인지 부하 | 1분 루틴은 한 화면, 한 문장, 하나의 진행 상태로 보여 준다. |
| 죄책감 없음 | 미루기·넘기기에 경고, 점수 하락, 설득 문구를 사용하지 않는다. |
| 맥락 존중 | PC는 전체 화면·회의·화면 공유 중 알림을 보류한다. Android는 사용자가 정한 업무 시간과 일시정지를 경계로 삼는다. |
| 알람의 명시성 | 다음 알람 시각, 반복 요일, 간격, 켜짐 상태를 홈과 설정에서 분명하게 보여 준다. |
| 개인정보 최소화 | 필요한 설정만 기기에 저장하며 외부 전송과 계정 가입을 하지 않는다. |
| 선택형 성취감 | 기록 기능은 명시적으로 켠 사람에게만 제공하고, 끈 사람의 O/X 응답은 저장하지 않는다. |
| 정직한 수치 | 물의 양을 입력받지 않으므로 `몇 mL 마심`이 아니라 `1분의 틈을 챙긴 횟수`만 표시한다. |

### 3.1 상반된 사용자 니즈

| 유형 | 실제 니즈 | 제품 대응 |
|---|---|---|
| 가볍게 쓰는 사람 | 체크·통계·달력은 귀찮고 알림만 필요하다. | 기본값을 `가볍게 사용`으로 둔다. 세 버튼으로 알림만 끝내며 결과는 저장하지 않는다. |
| 기록을 좋아하는 사람 | 오늘 몇 번 실천했는지 보고 뿌듯함을 느끼고 싶다. | 설정에서 `기록 남기기`를 켜면 O/X 결과를 기기 안에 저장하고 달력·시간대별 상세를 제공한다. |

기록 모드는 별도 앱이나 복잡한 전문가 모드가 아니다. 같은 알림 경험에 **로컬 기록과 달력만 더하는 선택 설정**이다.

## 4. 공통 기능 명세

### 4.1 업무 시간과 반복 주기

- 기본 업무 시간: 평일 오전 9시 ~ 오후 6시
- 기본 반복 주기: 60분
- 사용자는 업무 시작·종료 시간과 요일을 변경할 수 있다.
- 알림 간격: PC는 60분 고정. **Android는 15분~4시간 범위에서 5분 단위로 조절할 수 있다** (기본 60분). 프리셋(30분/45분/1시간/1시간 30분/2시간)과 스테퍼를 함께 제공한다.
- 업무 시간 밖에는 알림을 보내지 않는다. 업무 시간이 지나 다음 근무일로 넘어가면, 다음 알림은 **다음 근무일 업무 시작 + 간격** 시점이다.
- 업무 시작/종료/요일/간격 중 하나라도 바꾸면 다음 알림 시각을 즉시 다시 계산한다. 이전 설정으로 잡힌 예약을 남기지 않는다.
- PC는 사용자가 1분 루틴을 끝낸 시점부터 60분 뒤를 다음 알림 시점으로 잡는다.
- Android는 사용자가 O(`네, 챙겼어요`)를 누른 시점부터 설정한 간격 뒤를 다음 알림 시점으로 잡는다.
- 사용자가 `5분 뒤에 다시`를 선택하면 5분 뒤 한 번만 다시 알린다.
- 사용자가 X(`이번엔 넘길게요`)를 선택하면, 다음 정규 알림은 현재 예정 시점에서 간격만큼 뒤에 보낸다. 반복 재촉은 하지 않는다.

#### Android 알림 종료 버튼

Android의 1분 화면은 다음 세 버튼으로만 끝낸다.

| 표시 | 버튼 문구 | 의미 | 기록 모드에서의 저장 |
|---|---|---|---|
| 시계 | `5분 뒤에 다시` | 아직 판단하지 않고 같은 회차를 한 번 미룬다. | 저장하지 않음 |
| O | `네, 챙겼어요` | 물 한 모금 또는 스트레칭 중 하나 이상을 실천했다. | `done`과 실제 응답 시각 저장 |
| X | `이번엔 넘길게요` | 이번 회차는 실천하지 않고 종료한다. | `skipped`와 실제 응답 시각 저장 |

- O/X는 성공·실패 점수가 아니라 사용자의 자기 보고다.
- 워딩 선정 근거: `완료`는 숙제처럼 느껴지고 `못 했어요`는 죄책감을 줄 수 있어 응답 문구로 쓰지 않는다. 물과 스트레칭 중 무엇을 했는지는 추가로 묻지 않고, 하나라도 했다면 O로 기록한다.
- X는 빨간색·경고색을 사용하지 않고 중립 버튼으로 표시한다.
- 시스템 뒤로 가기, 강제 종료, 아무 응답 없이 사라진 경우는 X가 아니라 `응답 없음`으로 구분한다.
- `5분 뒤에 다시`는 같은 회차를 유지하며, 이후 O/X를 선택했을 때 최종 결과 하나만 남긴다.
- 기록 모드가 꺼져 있으면 O/X는 다음 알림 계산에만 사용하고 달력 데이터는 만들지 않는다.

### 4.2 알림 방식

알림 방식은 사용자 설정으로 무음 또는 세 단계 진동 중 하나를 선택한다.

| 모드 | 동작 | 적합한 상황 |
|---|---|---|
| 무음 | 소리와 진동 없이 시각 알림만 제공한다. | 회의, 공동 업무 공간, 높은 몰입 상태 |
| 가볍게 | 진동 1번과 시각 알림을 제공한다. | 짧게만 인지하고 싶은 경우 |
| 또렷하게 | 진동 3번과 시각 알림을 제공한다. | 일반적인 업무 환경 |
| 확실하게 | 진동 5번과 시각 알림을 제공한다. | 휴대폰이 조금 떨어져 있는 경우 |

- 기본값: **무음**
- 진동은 길이가 아니라 **횟수**로 구분한다. 한 펄스는 350ms 진동 + 250ms 쉼이며, "둥·둥·둥" 리듬으로 이어진다.
- **기기가 무음(벨소리 음소거) 모드여도 진동 모드를 선택했다면 진동이 전달된다.** 알람 앱과 같은 알람 우선순위 진동을 사용하기 때문이며, 사용자가 앱에서 `무음`을 고른 경우에만 진동하지 않는다.
- 설정·온보딩에서 진동 방식을 누르는 순간 해당 패턴을 즉시 미리보기로 울려 준다.
- 벨소리·효과음은 MVP에서 제공하지 않는다.
- Android의 시스템 알림 권한이 없으면 알림을 보낼 수 없음을 설정 화면에서 알려 준다.
- PC는 무음 시각 알림만 제공하고, 진동 옵션은 Android에만 노출한다.

### 4.3 화면 노출 시간

알림은 확인을 강요하지 않되, 사용자가 인지할 만큼 충분히 보인다.

| 화면/상태 | 노출 시간 | 이후 동작 |
|---|---:|---|
| PC의 작은 알림 배너 | 12초 | 사라지고 메뉴바 아이콘만 은은하게 강조한다. |
| Android 일반 알림 대체 상태 | 사용자가 확인할 때까지 시스템 알림 영역에 유지 | 전체 화면 권한이 없을 때 사용한다. |
| PC 1분 루틴 창 | 60초 | 마지막 안내 후 완료 화면으로 전환한다. |
| Android 1분 루틴 화면 | 60초 타이머 + 응답 대기 | 60초 종료를 O로 간주하지 않고 `어땠나요?` 상태에서 선택을 기다린다. |

PC는 60초가 지나면 완료 상태로 전환한다. Android는 60초가 지나도 자동으로 O를 기록하지 않고 `어땠나요?` 상태에서 세 선택지를 유지한다.

### 4.4 알림 문구

기본 문구:

> 지금 멈추라는 뜻은 아니에요.  
> 마무리할 틈이 생기면, 1분만 움직여요.

루틴 시작 문구:

> 물 한 모금 마시고, 자리에서 일어나 가볍게 스트레칭해 보세요.

완료 문구:

> 충분해요. 다시 당신의 흐름으로 돌아가요.

피해야 할 문구:

- `지금 하지 않으면 건강에 나쁩니다.`
- `오늘 목표를 놓치고 있어요.`
- `휴식을 건너뛰었습니다.`
- `스트레칭을 완료하세요.`

## 5. PC 앱 기획

### 5.1 PC 앱의 역할

PC 앱은 업무 중 가장 가까운 접점이다. 사용자가 메인 화면을 자주 열도록 만드는 대신, **메뉴바에서 조용히 상태를 보여 주고 필요할 때만 짧게 개입**한다.

### 5.2 PC 기능

- 메뉴바/시스템 트레이 상주 아이콘
- 다음 알림까지 남은 시간 확인
- 업무 리듬 시작·일시정지·재개
- 무음 알림
- PC OS가 지원하는 경우 시스템 알림으로 표시
- 전체 화면, 화면 공유, 회의 중으로 판단되는 상황에서는 알림 배너 보류
- 보류된 알림은 작업 전환 또는 입력 공백 이후 한 번만 다시 제안
- 1분 안내 창
- 업무 시간, 요일, PC 알림 노출 설정

### 5.3 PC 알림 우선순위

1. 시스템이 전체 화면 또는 화면 공유 상태로 보이면 배너를 띄우지 않는다.
2. 사용자가 활발히 입력 중이면 12초 배너 대신 메뉴바 아이콘만 강조할 수 있다.
3. 창 전환, 입력 공백, 전체 화면 종료 등 비교적 자연스러운 전환이 감지되면 한 번만 배너를 띄운다.
4. 사용자가 응답하지 않으면 알림을 반복하지 않는다.
5. 다음 정규 주기까지는 메뉴바에서만 조용히 `휴식 제안 중` 상태를 표시한다.

### 5.4 PC 화면 설계서

#### P-01. 메뉴바 기본 상태

**목적:** 앱을 열지 않아도 다음 휴식의 존재를 조용히 인지하게 한다.

- 위치: OS 메뉴바 또는 시스템 트레이
- 아이콘: `TeuM` 심볼. 작은 여백이 느껴지는 단순한 형태를 사용한다.
- 기본 표현: 아이콘만 표시한다.
- 클릭 시 팝오버를 연다.
- 길게 보이는 숫자 카운트다운은 메뉴바를 어지럽힐 수 있으므로 기본 상태에서는 표시하지 않는다.

#### P-02. 메뉴바 팝오버

**목적:** 가장 적은 조작으로 상태를 확인하고 제어한다.

텍스트 구성:

```text
틈새움 TeuM
다음 틈까지 24분

[ 지금 1분 ]
[ 잠시 멈춤 ]

오늘의 업무 시간 09:00–18:00   [ 설정 ]
```

- 너비: 약 280~320px
- `지금 1분`: 즉시 1분 루틴 창을 연다.
- `잠시 멈춤`: 누르면 하단에 `언제 다시 시작할까요?`와 `30분 / 오늘 업무 종료까지 / 직접 선택`을 표시한다.
- 설정은 작은 텍스트 버튼으로 배치한다.
- 기록, 달성률, 순위 등 수치 평가는 표시하지 않는다.

#### P-03. PC 휴식 제안 배너

**목적:** 업무를 끊지 않고 휴식할 선택권을 제공한다.

텍스트 구성:

```text
틈새움
마무리할 틈이 생기면, 1분만 움직여요.

[ 지금 1분 ]  [ 5분 뒤 ]  [ 이번엔 넘기기 ]
```

- 위치: 우측 하단. OS 알림 영역과 겹치지 않게 여백을 둔다.
- 크기: 약 360~420px 너비, 한눈에 읽히는 높이
- 노출: 12초
- 버튼 순서: 긍정 행동인 `지금 1분`을 가장 왼쪽에 놓되, 강조색은 과도하지 않게 한다.
- 닫기(X): 배너 우측 상단에 제공한다. 닫기는 `이번엔 넘기기`와 동일하게 처리한다.
- 12초 동안 조작이 없으면 사라지며, 메뉴바 아이콘의 작은 강조 상태만 남는다.
- 모달, 전체 화면 오버레이, 자동 포커스 이동은 사용하지 않는다.

#### P-04. PC 1분 루틴 창

**목적:** 사용자가 별도 판단 없이 잠깐 몸을 움직인 뒤 업무로 돌아가게 한다.

텍스트 구성:

```text
1분의 틈

물 한 모금 마시고,
자리에서 일어나 가볍게 스트레칭해 보세요.

남은 시간 00:42

[ 지금은 그만 ]
```

- 위치: 화면 중앙이 아닌 작업창을 크게 가리지 않는 중앙 하단 또는 사용자가 마지막으로 둔 위치
- 크기: 약 360~440px 너비
- 타이머: 60초에서 0초까지 부드럽게 감소. 초 단위 숫자만 지나치게 강조하지 않는다.
- 안내: 한 문장만 표시하고, 중간에 동작 지시를 추가하지 않는다.
- `지금은 그만`: 루틴을 종료하고 팝오버 상태로 돌아간다. 실패·경고 문구를 보여 주지 않는다.
- 60초 후: 완료 화면으로 전환한다.

#### P-05. PC 완료 상태

텍스트 구성:

```text
충분해요.
다시 당신의 흐름으로 돌아가요.

다음 틈은 약 1시간 뒤에 알려드릴게요.
```

- 노출: 4초
- 이후: 루틴 창을 닫고 메뉴바 기본 상태로 돌아간다.
- 완료 시점에 다음 60분 타이머를 시작한다.

#### P-06. PC 설정

섹션 구성:

```text
업무 시간
시작 [09:00]   종료 [18:00]
반복 요일 [월 화 수 목 금]

알림
알림 표시 [켜짐]
배너 표시 시간 [12초]
전체 화면·화면 공유 중 알림 보류 [켜짐]

데이터 및 개인정보
틈새움은 건강·행동 기록을 수집하거나 외부로 전송하지 않습니다.
```

- 알림 간격은 매 60분으로 고정한다. 사용자가 바꿀 수 있는 설정은 업무 시간과 요일뿐이다.
- PC 알림은 무음만 제공하므로 진동 관련 설정을 표시하지 않는다.
- 설정 변경은 즉시 반영한다.

## 6. Android 앱 기획

### 6.1 Android 앱의 역할

Android 앱은 PC와 독립적으로 작동하는 **직장인 건강 반복 알람**이다. 사용자는 알람 일정과 다음 실행 시각을 직접 확인·제어하며, 알람이 울렸을 때 `1분의 틈` 화면을 사용한다.

### 6.2 Android 기능

- 사용자가 정한 업무 시간·요일·간격의 반복 건강 알람 (15분~4시간, 기본 60분)
- 알림 모드: 무음 / 진동 1·3·5회 펄스 (기기 무음 모드에서도 진동 전달, 선택 시 미리보기)
- 잠금 화면·알림 영역의 빠른 액션과 전체 화면 1분 화면
- 1분 텍스트 안내 화면
- 홈 화면 위젯
- 업무 리듬 시작·일시정지·재개 (재개 시 다음 알림 시각을 토스트로 안내)
- 업무 시간, 요일, 알림 간격, 알림 방식, 전체 화면 알림 권한 안내 — 모두 자체 디자인 선택 시트 사용
- 기기 내부 설정만 저장. 계정·서버 동기화 없음.
- 선택형 로컬 기록 모드와 월간 달력

### 6.3 Android 화면 설계서

#### A-01. 온보딩: 환영

**목적:** 서비스가 무엇을 하고 무엇을 하지 않는지 첫 화면에서 이해시킨다.

텍스트 구성:

```text
틈새움 TeuM
일하는 나를 위한 1분

업무 중 정해 둔 간격마다,
물 한 모금과 가벼운 스트레칭을 챙겨 드려요.

[ 건강 알람 설정하기 ]
```

- 한 화면으로 끝낸다.
- 회원가입, 이름 입력, 건강 설문은 제공하지 않는다.
- 하단에는 작은 텍스트로 `건강 데이터를 수집하지 않아요.`를 표시한다.

#### A-02. 온보딩: 시간 설정

텍스트 구성:

```text
건강 알람을 언제 울릴까요?

업무 시작  [오전 09:00]
업무 종료  [오후 06:00]
알람 간격  [1시간]
반복 요일  [월 화 수 목 금]

[ 다음 ]
```

- 기본값은 평일 09:00~18:00.
- 시간 선택은 Android 표준 시계가 아니라 **자체 시간 선택 바텀시트**를 사용한다. 상단에 `오전 9:00` 미리보기, 시(±1)·분(±5) 스테퍼(길게 누르면 연속 증감), `취소/확인` 버튼으로 구성하며 앱의 파사드 디자인을 그대로 따른다.
- 사용자가 주말을 선택하지 않아도 진행 가능하다.

#### A-03. 온보딩: 알림 방식

텍스트 구성:

```text
건강 알람을 어떻게 알려드릴까요?

( ) 무음 — 화면으로만 조용히 알려드려요
( ) 가볍게 — 진동 1번
( ) 또렷하게 — 진동 3번
( ) 확실하게 — 진동 5번

[ 건강 알람 시작 ]
```

- 기본 선택: 무음
- 사용자가 `건강 알람 시작`을 누를 때 Android 알림 권한을 요청한다.
- 권한 요청 직전 설명: `정해 둔 업무 시간에 반복 알람을 울리고, 잠금 화면에서도 1분의 틈을 바로 보여 드려요.`
- 권한 거부 시에도 앱은 열리지만, 홈 화면에 `알림 권한을 켜야 건강 알람을 받을 수 있어요` 안내를 제공한다.

#### A-04. 홈: 진행 중

**목적:** 현재 리듬을 한눈에 보여 주고, 과도한 정보 없이 제어한다.

화면 구성 (위에서 아래로):

```text
[헤더] 틈새움(라벤더 필 로고) ─────────── [설정]

┌ NEXT HEALTH ALARM ─────────────┐
│ 다음 건강 알람까지                 │
│ 24분                             │
│ 물 한 모금과 가벼운 스트레칭을       │
│ 챙길 시간이에요.                   │
│ ▓▓▓▓▓▓▓░░░ (진행 미터)           │
└──────────────────────────────┘

┌ TODAY'S RHYTHM ─────────────────┐
│ 오늘의 업무 시간      09:00–18:00 │
│ 알림 방식      또렷하게 · 진동 3번  │
│ 현재 상태        건강 알람 대기 중  │
└──────────────────────────────┘

┌ CONTROL PANEL ──────────────────┐
│ 지금은 건강 알람을 받고 있어요.      │
│ [ 잠시 멈춤 ]                    │
└──────────────────────────────┘
```

- 히어로 플레이트는 고정 높이를 쓰지 않고 내용에 맞춰 끝난다. 빈 배경을 남기지 않는다.
- 업무 시간 밖에는 카운트다운 대신 **다음 알람의 실제 시각**을 크게 보여 준다: `다음 건강 알람 / 내일 오전 10:00` (같은 날은 시각만, 다음 날은 `내일`, 그 이후는 요일을 붙인다). 진행 미터는 숨긴다.
- 남은 시간 숫자는 tabular numerals로 표시해 흔들리지 않게 한다. 조급함을 유발하는 애니메이션·경고색은 쓰지 않는다.
- `잠시 멈춤`: 바텀시트로 `30분 / 오늘 업무 종료까지 / 직접 선택`을 제공한다. `직접 선택`은 자체 시간 선택 시트를 연다.
- 토스트(스낵바)는 하단 중앙에 **카본 배경 + 앰버 테두리의 필(pill)** 형태로 띄운다. 패널과 같은 폭·모양을 쓰면 겹쳐 보이므로 금지한다.
- `가볍게 사용`에서는 오늘 완료 횟수, 물 섭취량, 연속 일수를 보여 주지 않는다.
- `기록 남기기`가 켜져 있으면 홈 하단에 `오늘 3번 챙겼어요` 한 줄과 `기록 보기` 진입만 추가한다.
- `기록 보기`는 A-11 달력으로 이동한다. 기록 모드가 꺼져 있으면 홈과 별도 내비게이션에 기록 진입을 노출하지 않는다.

#### A-05. 홈: 일시정지 상태

텍스트 구성:

```text
지금은 쉬어 가는 중이에요.

건강 알람이 잠시 멈춰 있어요.

[ 다시 시작 ]
```

- 일시정지 종료 시각이 있으면 `오후 2:30에 다시 알려드릴게요.`를 함께 표시한다.
- 사용자가 직접 재개할 수 있다.
- `다시 시작`을 누르면 토스트로 **다음 알람의 실제 시각**을 알려 준다: `다시 시작할게요. 다음 건강 알람은 오후 3:30이에요.` (다음 날이면 `내일 오전 10:00`처럼 홈 히어로와 같은 표기를 쓴다). "약 1시간 뒤" 같은 모호한 상대 표현은 쓰지 않는다.

#### A-06. Android 알림

알림 제목과 본문:

```text
틈새움 건강 알람
물 한 모금과 가벼운 스트레칭을 챙길 시간이에요.
```

알림이 열리면 1분 화면에 다음 액션을 제공한다.

```text
[ 5분 뒤에 다시 ] [ O 네, 챙겼어요 ] [ X 이번엔 넘길게요 ]
```

- 화면이 켜져 있거나 잠겨 있어도 권한이 허용되면 1분 화면을 바로 연다.
- 전체 화면 알림 권한이 없으면 높은 우선순위의 일반 알림으로 대체한다.
- 무음 모드: 소리·진동 없음.
- 진동 모드: 설정한 1회·3회·5회 패턴을 한 번만 실행한다.
- 같은 알림을 반복 송신하지 않는다.

#### A-07. Android 1분 루틴 및 응답

텍스트 구성:

```text
업무 건강 알람
1분의 틈

물 한 모금 마시고,
자리에서 일어나 가볍게 스트레칭해 보세요.

물 한 모금이나 스트레칭,
하나라도 챙겼나요?

00:42

[ 5분 뒤에 다시 ]
[ O  네, 챙겼어요 ] [ X  이번엔 넘길게요 ]
```

- 큰 글자와 높은 명도 대비를 사용한다.
- 알림으로 열린 60초 동안 화면을 켜 둔다.
- 이미지, 영상, 자세 체크, 운동 횟수 카운트는 제공하지 않는다.
- 질문 문구가 O의 기준을 설명한다. 물과 스트레칭 중 하나 이상을 했다면 O로 응답할 수 있으며, 무엇을 했는지는 별도로 묻지 않는다.
- 세 선택지를 한 줄에 억지로 넣지 않는다. `5분 뒤에 다시`는 중립색 전체 폭 버튼으로 위에 두고, 아래 행에 `O 네, 챙겼어요`와 `X 이번엔 넘길게요`를 같은 크기로 배치한다.
- `네, 챙겼어요`를 누르면 A-08 완료 상태를 거쳐 홈으로 돌아가고, 기록 모드에서는 해당 시각에 O를 저장한다.
- `이번엔 넘길게요`를 누르면 즉시 홈으로 돌아가고, 기록 모드에서는 해당 시각에 X를 저장한다.
- `5분 뒤에 다시`는 화면을 닫고 같은 회차를 5분 뒤 다시 연다.
- 60초가 끝나도 자동으로 O 처리하지 않는다. 사용자가 명시적으로 O를 눌러야 `챙김`으로 기록한다.
- O/X를 누르지 않고 화면을 닫은 경우 `응답 없음`이며 완료 횟수에 포함하지 않는다.

#### A-08. Android O 응답 후 완료 상태

텍스트 구성:

```text
충분해요.
오늘의 건강한 1분을 챙겼어요.
```

- 노출: 4초
- 이후 홈 화면으로 이동한다.
- O를 누른 시점부터 다음 60분을 계산한다.
- X는 기존 정규 주기를 유지한다.

#### A-09. 홈 화면 위젯

**목적:** 앱을 열지 않고도 상태를 확인·제어하게 한다.

작은 위젯:

```text
TeuM
다음 건강 알람까지 24분
```

큰 위젯:

```text
틈새움
다음 건강 알람까지 24분
물 한 모금과 가벼운 스트레칭

[ 잠시 멈춤 ]
```

- 위젯은 완료 횟수나 건강 수치를 표시하지 않는다.
- `잠시 멈춤`을 누르면 30분 일시정지를 즉시 적용하고, 가벼운 토스트로만 알린다.

#### A-10. Android 설정

섹션 구성:

```text
건강 알람 일정
시작 [09:00]   종료 [18:00]
알림 간격 [1시간]
반복 요일 [월 화 수 목 금]

건강 알람
알람 [켜짐]
방식 [무음 / 가볍게·진동 1번 / 또렷하게·진동 3번 / 확실하게·진동 5번]

개인정보
틈새움은 정보를 서버로 수집하거나 외부로 전송하지 않습니다.
기록 모드의 O/X 결과는 이 기기에만 저장됩니다.
[ 개인정보처리방침 보기 ↗ ]
```

- 시작·종료·알림 간격 행을 누르면 자체 선택 시트가 열린다. 간격 시트는 프리셋 칩(30분~2시간)과 `−15/−5/+5/+15` 스테퍼를 제공하며 15분~4시간, 5분 단위다.
- 진동 방식을 누르면 해당 패턴을 즉시 미리보기로 울린다.
- 시간·간격·요일 변경은 즉시 저장되고 다음 알림 시각을 다시 계산한다.
- 알림 권한이 꺼져 있으면 시스템 설정으로 가는 버튼을 표시한다. 전체 화면 알림 권한이 없을 때만 `권한 필요` 패널과 허용 버튼을 노출한다.
- **새 기능의 설명·자랑을 설정 화면에 두지 않는다.** 화면에는 사용자가 조작할 것과, 지금 조치가 필요한 권한 안내만 남긴다.
- 모든 설정은 기기 내에만 저장한다.

기록 섹션을 추가한다.

```text
사용 방식
(●) 가볍게 사용 — 응답을 저장하지 않아요
( ) 기록 남기기 — O/X 결과를 이 기기에 저장해요

기록 관리
[ 기록 모두 삭제 ]
```

- 기본값은 `가볍게 사용`이다.
- `기록 남기기`를 켠 순간부터 새 결과만 저장한다. 이전의 미기록 회차를 추정해서 채우지 않는다.
- 기록 모드를 꺼도 기존 기록은 자동 삭제하지 않고 달력만 숨긴다. 다시 켜면 이전 기록을 볼 수 있다.
- 삭제는 확인 다이얼로그를 한 번 거치며 되돌릴 수 없음을 명확히 알린다.

#### A-11. 기록 달력

**목적:** 목표 압박 없이, 사용자가 스스로 챙긴 횟수를 월 단위로 돌아보게 한다.

```text
나의 틈 기록                 2026년 7월
                       [ ‹ ] [ › ]

일  월  화  수  목  금  토
        1   2   3   4
        3회 4회 2회 5회
 5   6   7   8   9  10  11
     1회 4회 3회     2회

오늘 4번 챙겼어요.
```

- 달력 날짜 셀에는 O(`챙겼어요`) 횟수만 `N회`로 표시한다.
- X와 응답 없음은 날짜 셀 숫자에 포함하지 않는다.
- 목표 횟수, 성공률, 연속 일수, 불꽃 아이콘, 미달 경고는 표시하지 않는다.
- 기록이 없는 날은 `0회` 대신 빈칸으로 두어 실패처럼 보이지 않게 한다.
- 날짜를 누르면 A-12 시간대별 상세로 이동한다.
- 기록은 Android 기기 안에만 있으며 PC 기록과 합산하지 않는다.

디자인 적용 (현행 파사드 언어 기준):

- 달력 전체는 카본 라벨 탭(`나의 틈 기록`)이 붙은 플래티넘 패널 하나로 만든다. 요일 헤더는 크롬 인디고 소문자 라벨, 날짜 그리드 셀은 화이트 인셋(1px 헤어라인)으로 구성한다.
- 오늘 날짜는 앰버 배경 칩으로만 구분한다. O 횟수 `N회`는 크롬 인디고 볼드 12px — 색이 아니라 텍스트가 정보를 전달한다.
- 월 이동 `‹ ›`는 시간 선택 시트와 같은 베벨 스테퍼 버튼(48dp)을 재사용한다.
- 요일 칩·간격 칩과 마찬가지로 선택·강조는 앰버, 비활성은 화이트+뮤티드 인디고로 통일한다. 빨강·초록 등 달력 앱 관례색은 쓰지 않는다.

#### A-12. 날짜별 시간대 기록

```text
7월 8일 수요일
오늘 4번 챙겼어요.

09:58  O  챙겼어요
11:03  X  이번 틈은 넘겼어요
12:08  O  챙겼어요
14:11  —  응답하지 않았어요
15:06  O  챙겼어요
17:02  O  챙겼어요
```

- O/X는 사용자가 선택한 실제 시각을 표시한다. `응답 없음`은 화면을 닫은 시각을 표시한다.
- `5분 뒤에 다시`는 별도 행으로 쌓지 않고, 최종 O/X 행에 `5분 미룸` 보조 표지만 붙일 수 있다.
- O는 시그널 오렌지 또는 앰버, X와 응답 없음은 중립색으로 표시한다. 빨간색은 사용하지 않는다.
- 행 수정·개별 삭제는 MVP에서 제공하지 않는다. 기록 삭제는 설정의 `기록 모두 삭제`에서만 제공한다.
- 물을 마신 양을 입력받지 않으므로 `오늘 800mL` 같은 수치는 표시하지 않는다.

### 6.4 로컬 기록 데이터 규칙

기록 모드에서 저장하는 최소 필드:

```text
id            회차 식별자
scheduledAt   최초 알림 예정 시각
resolvedAt    O/X 선택 또는 응답 없이 닫힌 실제 시각
result        done | skipped | unanswered
snoozed       5분 미루기 사용 여부
```

- 서버, 계정, 광고·분석 SDK로 전송하지 않는다.
- 날짜와 시간대 표시는 현재 기기 시간대를 사용한다.
- 같은 회차는 최종 결과 하나만 저장해 중복 집계를 막는다.
- 시스템 뒤로 가기나 화면 닫기로 종료한 회차는 즉시 `unanswered`로 확정한다. 앱 강제 종료처럼 종료 시점을 확정할 수 없으면 다음 실행 때 마지막 확인 시각을 사용한다.
- 앱 데이터 삭제 또는 `기록 모두 삭제` 시 영구 삭제한다.
- 달력 집계는 저장된 `done`을 날짜별로 세는 단순 계산이며 건강 효과를 분석하지 않는다.

### 6.5 Android 디자인 적용 기준 (현행)

`DESIGN-nintendo-2001.md`의 2001년 콘솔 파사드 언어를 따르되, 모바일·건강 앱 맥락에 맞게 다음과 같이 조정해 적용 중이다.

| 시그널 오렌지 | 원본 `#f68d1f`는 장시간 보기에 자극적이라 `#e2954f`로 완화했고, 주황 버튼 글자는 흰색 대신 카본을 쓴다. 브랜드 빨강은 화면에서 쓰지 않는다. |

| 라운드 | 원칙적으로 각지게. 라운드는 로고 필, 라디오, 하단 토스트 필에만 허용한다. |
| 토스트 | 하단 중앙, 카본 배경 + 앰버 테두리 필. 콘텐츠 패널과 같은 폭·형태 금지. |
| 시간·간격 선택 | OS 기본 다이얼 대신 자체 바텀시트(화이트 디지트 플레이트 + 베벨 스테퍼)로 파사드 언어를 유지한다. |
| 타이포 | 큰 숫자는 tabular numerals. 상태·정보는 색+텍스트 병행. 경고색·깜빡임 금지. |
| 상태바·제스처 영역 | 상태바 높이만큼 카본으로 채워 헤더와 한 덩어리로 보이게 하고, 푸터는 제스처 바와 겹치지 않게 여백을 확보한다. |

### 6.6 Android 기술 구현 노트 (현행)

| 영역 | 구현 |
|---|---|
| 정규 알림 예약 | `AlarmManager.setExactAndAllowWhileIdle` + `SCHEDULE_EXACT_ALARM` 권한. 권한이 거부되면 비정확 예약으로 폴백하고 지연 가능성을 안내한다. 예약은 항상 1건만 유지하며, 상태가 바뀔 때마다 취소 후 재예약한다. |
| 1분 화면 열기 | 알람 수신 리시버가 전체 화면 인텐트(`USE_FULL_SCREEN_INTENT`) 알림을 게시한다. 화면 사용 중이면 즉시 열기를 시도하고, 실패하면 헤드업 알림이 대체한다. 스토어 제출 전 notification trampoline 정책 재검토가 필요하다(배포 문서 §2). |
| 무음 모드 진동 | 알림 채널 진동은 기기 벨소리 모드에 종속되므로 사용하지 않는다(전 채널 `enableVibration=false`). 대신 리시버가 `Vibrator`를 `VibrationAttributes.USAGE_ALARM`으로 직접 호출해, 기기가 무음이어도 사용자가 고른 1·3·5회 펄스가 전달된다. 채널과 직접 호출의 이중 진동은 구조적으로 발생하지 않는다. |
| 진동 패턴 | 펄스당 350ms 진동 + 250ms 쉼. 패턴 정의는 네이티브 `AlarmVibration.patternFor` 한 곳에 두고 JS·리시버·미리보기가 공유한다. |
| 알림 채널 | 채널 설정은 생성 후 코드로 변경할 수 없으므로, 설정을 바꿀 때는 채널 ID 버전을 올려 재생성하고 구 채널을 삭제한다. 채널 사운드는 `null`(기본음 아님)로 명시한다. |
| 상태 저장 | AsyncStorage 단일 JSON(설정+리듬). 서버·계정 없음. 놓친 알림 시각은 실행 시 다음 정규 슬롯으로 롤포워드한다. |
| 설정 변경 재계산 | 시작/종료/요일/간격 중 하나라도 바뀌면 `nextTickAt`을 현재 시각 기준으로 재계산하고 네이티브 알람을 다시 잡는다. |

## 7. 상태 및 알림 흐름

```text
[업무 시간 시작]
        ↓
[60분 대기]
        ↓
[휴식 제안]
   └─ [1분 텍스트 루틴]
        ├─ 5분 뒤에 다시       → [같은 회차 5분 대기] → [1분 화면 1회]
        ├─ O 네, 챙겼어요       → [선택 시각부터 60분 대기]
        └─ X 이번엔 넘길게요    → [다음 정규 알림까지 대기]
```

추가 규칙:

- `5분 뒤` 재알림에도 반응이 없으면 더 이상 재촉하지 않는다.
- 업무 종료 시각을 지나면 모든 보류·재알림을 취소한다.
- 일시정지 중에는 루틴 화면 및 예약 알림을 열지 않는다.
- 기록 모드에서는 O/X만 저장하며, 미루기는 같은 회차의 보조 상태로 남긴다.

## 8. 접근성 및 표현 기준

- 모든 핵심 버튼은 손가락으로 누르기 쉬운 충분한 크기로 제공한다.
- 색만으로 상태를 구분하지 않고 텍스트를 함께 표시한다.
- Android 응답 버튼의 명칭을 일관되게 유지한다: `5분 뒤에 다시`, `네, 챙겼어요`, `이번엔 넘길게요`.
- O/X 기호만으로 의미를 전달하지 않고 반드시 문구를 함께 표시한다.
- 진동을 켠 사용자는 시각 정보도 함께 받아야 하며, 진동만으로 의미를 전달하지 않는다.
- 타이머는 초조함을 유발하는 경고색·깜빡임·큰 소리를 사용하지 않는다.
- 사용자가 시스템의 글자 크기를 키워도 핵심 문장과 3개 버튼이 잘리지 않게 설계한다.

## 9. MVP 출시 기준

다음 조건을 만족하면 MVP를 출시할 수 있다.

- PC는 60분 주기, Android는 설정한 간격(기본 60분)으로 업무 시간 안에 알림이 작동한다.
- Android에서 무음/진동 1·3·5회를 선택할 수 있고, 진동 모드는 기기가 무음이어도 전달된다.
- PC 알림 배너는 12초 노출된다. Android는 OS가 허용하면 1분 화면을 열고, 허용하지 않으면 일반 알림을 사용자가 확인할 때까지 유지한다.
- Android 1분 화면에서 `5분 뒤에 다시`, `네, 챙겼어요`, `이번엔 넘길게요`를 사용할 수 있다.
- 루틴 화면은 60초 동안 텍스트만 안내한다.
- O를 선택한 시점부터 사용자가 설정한 간격으로 다음 건강 알람을 계산한다.
- 기록 모드는 기본값이 아니며, 켠 경우에도 O/X 자기 보고 기록을 기기에만 저장하고 서버로 전송하지 않는다.
- 기록 달력에서 날짜별 O 횟수와 시간대별 O/X 결과를 볼 수 있다.
- 사용자는 언제든 알림을 일시정지하거나 끌 수 있다.
- 홈을 처음 본 사용자가 다음 건강 알람 시각과 알람 켜짐 상태를 10초 안에 찾을 수 있다.
- 스토어 대표 이미지와 온보딩을 본 사용자가 틈새움을 `직장인 스트레칭·물 마시기 건강 앱`으로 이해한다.

## 10. 이후 검토할 항목 (MVP 제외)

- PC와 Android 간 설정 동기화 여부
- 회사용 배포 및 조직 공통 업무 시간 설정
- 사용자가 선택할 수 있는 다른 휴식 문구
- 화면 공유·회의 상태를 더 정확히 감지하는 범위와 권한
- 건강 관련 안내 문구의 전문가 검토
- 물/스트레칭을 각각 구분해 기록할지 여부. 한 번의 추가 탭도 피로를 만들 수 있으므로 MVP에서는 합친 `1분의 틈`만 기록한다.

이 항목들은 기본 경험이 안정화된 뒤 검토하며, 서버 수집을 전제로 하지 않는다.
