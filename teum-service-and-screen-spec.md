# 틈새움 TeuM 서비스 기획 및 화면 설계서

> 버전: MVP 1.0  
> 범위: PC 앱, Android 앱  
> 원칙: 개인 건강 데이터를 수집·저장·분석하지 않는다.

## 1. 서비스 정의

### 이름

- 한글명: **틈새움**
- 영문명: **TeuM**
- 앱 아이콘 표기: **TeuM**
- 스토어 표기: **틈새움 TeuM**
- 영문 의미: *A small pause that keeps your flow moving.*

### 한 줄 소개

**일의 흐름을 깨지 않으면서, 매시간 1분간 물을 마시고 몸을 펴도록 알려 주는 직장인용 리듬 앱.**

### 해결하려는 문제

직장인은 장시간 앉아 일하면서 물 마시기와 짧은 움직임을 자주 놓친다. 그러나 알림이 잦거나, 강제적이거나, 업무 중 화면을 크게 가리면 오히려 집중을 해친다.

틈새움은 건강을 감시하거나 사용자를 평가하지 않는다. 업무 중 자연스러운 마무리 지점에 맞춰 **짧고 부담 없는 1분의 틈**을 제안한다.

### MVP의 단일 루틴

알림을 받으면 사용자는 다음을 한다.

1. 물을 몇 모금 마신다.
2. 자리에서 일어나 몸을 편다.
3. 1분 뒤 업무로 돌아간다.

앱은 각 동작을 영상·그림·카운트로 강제하지 않는다. 화면에는 짧은 문장으로만 안내한다.

> 물 한 모금 마시고, 자리에서 일어나 몸을 가볍게 펴 보세요.

## 2. 목표와 비목표

### 목표

- 업무 시간에 1시간 간격으로 1분 휴식의 계기를 만든다.
- 알림을 최소한의 방해로 전달한다.
- 사용자가 쉽게 미루고, 쉽게 다시 시작할 수 있게 한다.
- PC와 Android에서 같은 루틴을 서로 다른 사용 맥락에 맞게 제공한다.

### 비목표

- 걸음 수, 수분 섭취량, 체중, 심박수, 자세, 건강 상태를 기록하지 않는다.
- 휴식 완료율, 연속 기록, 랭킹, 주간 리포트 등 행동 데이터를 모으지 않는다.
- 카메라·마이크·위치·연락처·접근성 권한을 사용하지 않는다.
- 의학적 진단, 치료, 건강 효과를 약속하지 않는다.
- 스트레칭 자세를 평가하거나 정확한 동작 수행을 요구하지 않는다.

## 3. HCI 핵심 원칙

| 원칙 | 제품 적용 |
|---|---|
| 방해 최소화 | 한 시간에 알림 하나만 보낸다. 업무 화면을 강제로 잠그지 않는다. |
| 사용자의 주도권 | `지금 하기`, `5분 뒤`, `이번엔 넘기기`를 항상 제공한다. |
| 작은 결정 | 알림에서 선택지는 최대 3개로 제한한다. |
| 낮은 인지 부하 | 1분 루틴은 한 화면, 한 문장, 하나의 진행 상태로 보여 준다. |
| 죄책감 없음 | 미루기·넘기기에 경고, 점수 하락, 설득 문구를 사용하지 않는다. |
| 맥락 존중 | 전체 화면·회의·화면 공유 등 방해하기 나쁜 상황에서는 강한 알림을 보류한다. |
| 개인정보 최소화 | 필요한 설정만 기기에 저장하며 외부 전송과 계정 가입을 하지 않는다. |

## 4. 공통 기능 명세

### 4.1 업무 시간과 반복 주기

- 기본 업무 시간: 평일 오전 9시 ~ 오후 6시
- 기본 반복 주기: 60분
- 사용자는 업무 시작·종료 시간과 요일을 변경할 수 있다. 알림 간격은 60분으로 고정한다.
- 업무 시간 밖에는 알림을 보내지 않는다.
- 사용자가 `지금 1분` 루틴을 끝내면, **완료 시점부터 60분 뒤**를 다음 알림 시점으로 잡는다.
- 사용자가 `5분 뒤`를 선택하면 5분 뒤 한 번만 다시 알린다.
- 사용자가 `이번엔 넘기기`를 선택하면, 다음 정규 알림은 현재 예정 시점에서 60분 뒤에 보낸다. 반복 재촉은 하지 않는다.

### 4.2 알림 방식

알림 방식은 사용자 설정으로 둘 중 하나를 선택한다.

| 모드 | 동작 | 적합한 상황 |
|---|---|---|
| 진동 | 짧은 진동과 시각 알림을 제공한다. 소리는 사용하지 않는다. | 휴대폰을 가까이 두고 일하는 경우 |
| 무음 | 소리와 진동 없이 시각 알림만 제공한다. | 회의, 공동 업무 공간, 높은 몰입 상태 |

- 기본값: **무음**
- 벨소리·효과음은 MVP에서 제공하지 않는다.
- Android의 시스템 알림 권한이 없으면 알림을 보낼 수 없음을 설정 화면에서 알려 준다.
- PC에서는 OS 알림 설정에 따라 진동이 없을 수 있으므로, `진동` 옵션은 Android에만 노출한다.

### 4.3 화면 노출 시간

알림은 확인을 강요하지 않되, 사용자가 인지할 만큼 충분히 보인다.

| 화면/상태 | 노출 시간 | 이후 동작 |
|---|---:|---|
| PC의 작은 알림 배너 | 12초 | 사라지고 메뉴바 아이콘만 은은하게 강조한다. |
| Android 잠금·상단 알림 | 사용자가 확인할 때까지 시스템 알림 영역에 유지 | 알림을 열거나 액션을 누르면 처리한다. |
| Android 화면 상단 헤드업 | 약 8초 | 상단 배너는 닫히며 알림 영역에는 남는다. |
| PC 1분 루틴 창 | 60초 | 마지막 안내 후 완료 화면으로 전환한다. |
| Android 1분 루틴 화면 | 60초 | 마지막 안내 후 완료 화면으로 전환한다. |
| 완료 화면 | 4초 | 자동으로 홈/이전 화면으로 돌아간다. |

시간이 끝나도 루틴 창을 자동으로 닫아 사용자를 놀라게 하지 않는다. 60초가 지나면 `다시 업무로 돌아가요` 상태로 바꾸고, 사용자가 `닫기`를 누르거나 4초 후 부드럽게 닫는다.

### 4.4 알림 문구

기본 문구:

> 지금 멈추라는 뜻은 아니에요.  
> 마무리할 틈이 생기면, 1분만 움직여요.

루틴 시작 문구:

> 물 한 모금 마시고, 자리에서 일어나 몸을 가볍게 펴 보세요.

완료 문구:

> 충분해요. 다시 당신의 흐름으로 돌아가요.

피해야 할 문구:

- `지금 하지 않으면 건강에 나쁩니다.`
- `오늘 목표를 놓치고 있어요.`
- `휴식을 건너뛰었습니다.`
- `스트레칭을 완료하세요.`

## 5. PC 앱 기획

### 5.1 PC 앱의 역할

PC 앱은 업무 중 가장 가까운 접점이다. 사용자가 메인 화면을 자주 열도록 만드는 대신, **메뉴바에서 조용히 상태를 보여 주고 필요할 때만 짧게 개입**한다.

### 5.2 PC 기능

- 메뉴바/시스템 트레이 상주 아이콘
- 다음 알림까지 남은 시간 확인
- 업무 리듬 시작·일시정지·재개
- 무음 알림
- PC OS가 지원하는 경우 시스템 알림으로 표시
- 전체 화면, 화면 공유, 회의 중으로 판단되는 상황에서는 알림 배너 보류
- 보류된 알림은 작업 전환 또는 입력 공백 이후 한 번만 다시 제안
- 1분 안내 창
- 업무 시간, 요일, PC 알림 노출 설정

### 5.3 PC 알림 우선순위

1. 시스템이 전체 화면 또는 화면 공유 상태로 보이면 배너를 띄우지 않는다.
2. 사용자가 활발히 입력 중이면 12초 배너 대신 메뉴바 아이콘만 강조할 수 있다.
3. 창 전환, 입력 공백, 전체 화면 종료 등 비교적 자연스러운 전환이 감지되면 한 번만 배너를 띄운다.
4. 사용자가 응답하지 않으면 알림을 반복하지 않는다.
5. 다음 정규 주기까지는 메뉴바에서만 조용히 `휴식 제안 중` 상태를 표시한다.

### 5.4 PC 화면 설계서

#### P-01. 메뉴바 기본 상태

**목적:** 앱을 열지 않아도 다음 휴식의 존재를 조용히 인지하게 한다.

- 위치: OS 메뉴바 또는 시스템 트레이
- 아이콘: `TeuM` 심볼. 작은 여백이 느껴지는 단순한 형태를 사용한다.
- 기본 표현: 아이콘만 표시한다.
- 클릭 시 팝오버를 연다.
- 길게 보이는 숫자 카운트다운은 메뉴바를 어지럽힐 수 있으므로 기본 상태에서는 표시하지 않는다.

#### P-02. 메뉴바 팝오버

**목적:** 가장 적은 조작으로 상태를 확인하고 제어한다.

텍스트 구성:

```text
틈새움 TeuM
다음 틈까지 24분

[ 지금 1분 ]
[ 잠시 멈춤 ]

오늘의 업무 시간 09:00–18:00   [ 설정 ]
```

- 너비: 약 280~320px
- `지금 1분`: 즉시 1분 루틴 창을 연다.
- `잠시 멈춤`: 누르면 하단에 `언제 다시 시작할까요?`와 `30분 / 오늘 업무 종료까지 / 직접 선택`을 표시한다.
- 설정은 작은 텍스트 버튼으로 배치한다.
- 기록, 달성률, 순위 등 수치 평가는 표시하지 않는다.

#### P-03. PC 휴식 제안 배너

**목적:** 업무를 끊지 않고 휴식할 선택권을 제공한다.

텍스트 구성:

```text
틈새움
마무리할 틈이 생기면, 1분만 움직여요.

[ 지금 1분 ]  [ 5분 뒤 ]  [ 이번엔 넘기기 ]
```

- 위치: 우측 하단. OS 알림 영역과 겹치지 않게 여백을 둔다.
- 크기: 약 360~420px 너비, 한눈에 읽히는 높이
- 노출: 12초
- 버튼 순서: 긍정 행동인 `지금 1분`을 가장 왼쪽에 놓되, 강조색은 과도하지 않게 한다.
- 닫기(X): 배너 우측 상단에 제공한다. 닫기는 `이번엔 넘기기`와 동일하게 처리한다.
- 12초 동안 조작이 없으면 사라지며, 메뉴바 아이콘의 작은 강조 상태만 남는다.
- 모달, 전체 화면 오버레이, 자동 포커스 이동은 사용하지 않는다.

#### P-04. PC 1분 루틴 창

**목적:** 사용자가 별도 판단 없이 잠깐 몸을 움직인 뒤 업무로 돌아가게 한다.

텍스트 구성:

```text
1분의 틈

물 한 모금 마시고,
자리에서 일어나 몸을 가볍게 펴 보세요.

남은 시간 00:42

[ 지금은 그만 ]
```

- 위치: 화면 중앙이 아닌 작업창을 크게 가리지 않는 중앙 하단 또는 사용자가 마지막으로 둔 위치
- 크기: 약 360~440px 너비
- 타이머: 60초에서 0초까지 부드럽게 감소. 초 단위 숫자만 지나치게 강조하지 않는다.
- 안내: 한 문장만 표시하고, 중간에 동작 지시를 추가하지 않는다.
- `지금은 그만`: 루틴을 종료하고 팝오버 상태로 돌아간다. 실패·경고 문구를 보여 주지 않는다.
- 60초 후: 완료 화면으로 전환한다.

#### P-05. PC 완료 상태

텍스트 구성:

```text
충분해요.
다시 당신의 흐름으로 돌아가요.

다음 틈은 약 1시간 뒤에 알려드릴게요.
```

- 노출: 4초
- 이후: 루틴 창을 닫고 메뉴바 기본 상태로 돌아간다.
- 완료 시점에 다음 60분 타이머를 시작한다.

#### P-06. PC 설정

섹션 구성:

```text
업무 시간
시작 [09:00]   종료 [18:00]
반복 요일 [월 화 수 목 금]

알림
알림 표시 [켜짐]
배너 표시 시간 [12초]
전체 화면·화면 공유 중 알림 보류 [켜짐]

데이터 및 개인정보
틈새움은 건강·행동 기록을 수집하거나 외부로 전송하지 않습니다.
```

- 알림 간격은 매 60분으로 고정한다. 사용자가 바꿀 수 있는 설정은 업무 시간과 요일뿐이다.
- PC 알림은 무음만 제공하므로 진동 관련 설정을 표시하지 않는다.
- 설정 변경은 즉시 반영한다.

## 6. Android 앱 기획

### 6.1 Android 앱의 역할

Android 앱은 PC와 독립적으로 작동하는 개인 휴식 도구다. 사용자는 홈 화면 위젯과 잠금 화면 알림으로 빠르게 상태를 확인하고, 필요할 때만 앱을 연다.

### 6.2 Android 기능

- 업무 시간 내 60분 반복 알림
- 알림 모드: 진동 또는 무음
- 잠금 화면·알림 영역의 빠른 액션
- 1분 텍스트 안내 화면
- 홈 화면 위젯
- 업무 리듬 시작·일시정지·재개
- 업무 시간, 요일, 알림 방식, 헤드업 노출 설정
- 기기 내부 설정만 저장. 계정·서버 동기화 없음.

### 6.3 Android 화면 설계서

#### A-01. 온보딩: 환영

**목적:** 서비스가 무엇을 하고 무엇을 하지 않는지 첫 화면에서 이해시킨다.

텍스트 구성:

```text
틈새움 TeuM
흐름을 지키는 작은 틈

업무 중 1시간마다,
물 한 모금과 1분의 움직임을 제안해요.

[ 시작하기 ]
```

- 한 화면으로 끝낸다.
- 회원가입, 이름 입력, 건강 설문은 제공하지 않는다.
- 하단에는 작은 텍스트로 `건강 데이터를 수집하지 않아요.`를 표시한다.

#### A-02. 온보딩: 시간 설정

텍스트 구성:

```text
언제 알려드릴까요?

업무 시작  [오전 09:00]
업무 종료  [오후 06:00]
반복 요일  [월 화 수 목 금]

[ 다음 ]
```

- 기본값은 평일 09:00~18:00.
- 시간 선택은 Android 표준 시간 선택기를 사용한다.
- 사용자가 주말을 선택하지 않아도 진행 가능하다.

#### A-03. 온보딩: 알림 방식

텍스트 구성:

```text
어떻게 알려드릴까요?

( ) 무음 — 화면으로만 조용히 알려드려요
( ) 진동 — 짧은 진동과 함께 알려드려요

[ 틈새움 시작 ]
```

- 기본 선택: 무음
- 사용자가 `틈새움 시작`을 누를 때 Android 알림 권한을 요청한다.
- 권한 요청 직전 설명: `정해 둔 업무 시간에만 1시간 간격으로 알려드려요.`
- 권한 거부 시에도 앱은 열리지만, 홈 화면에 `알림을 켜면 틈새움을 시작할 수 있어요` 안내를 제공한다.

#### A-04. 홈: 진행 중

**목적:** 현재 리듬을 한눈에 보여 주고, 과도한 정보 없이 제어한다.

텍스트 구성:

```text
틈새움
다음 틈까지
24분

물 한 모금과 1분의 움직임,
흐름을 해치지 않을 때 해요.

[ 지금 1분 ]

오늘의 업무 시간 09:00–18:00
[ 잠시 멈춤 ]                 [ 설정 ]
```

- 화면 중앙: 남은 시간. 원형 진행률은 선택 사항이며, 숫자보다 조급함을 유발하지 않도록 과도한 애니메이션을 사용하지 않는다.
- `지금 1분`: 즉시 루틴 화면으로 이동한다.
- `잠시 멈춤`: 바텀시트로 `30분 / 오늘 업무 종료까지 / 직접 선택`을 제공한다.
- 오늘 완료 횟수, 물 섭취량, 연속 일수는 보여 주지 않는다.

#### A-05. 홈: 일시정지 상태

텍스트 구성:

```text
지금은 쉬어 가는 중이에요.

알림이 잠시 멈춰 있어요.

[ 다시 시작 ]
```

- 일시정지 종료 시각이 있으면 `오후 2:30에 다시 알려드릴게요.`를 함께 표시한다.
- 사용자가 직접 재개할 수 있다.

#### A-06. Android 알림

알림 제목과 본문:

```text
틈새움
마무리할 틈이 생기면, 1분만 움직여요.
```

알림 액션:

```text
[ 지금 1분 ] [ 5분 뒤 ] [ 이번엔 넘기기 ]
```

- 헤드업 노출: 약 8초. 이후 알림 서랍에 유지한다.
- 잠금 화면에는 기기 설정이 허용하는 범위에서 제목·버튼을 표시한다.
- 무음 모드: 소리·진동 없음.
- 진동 모드: 짧은 단일 진동 패턴만 사용한다. 연속·강한 진동은 사용하지 않는다.
- 같은 알림을 반복 송신하지 않는다.

#### A-07. Android 1분 루틴

텍스트 구성:

```text
1분의 틈

물 한 모금 마시고,
자리에서 일어나 몸을 가볍게 펴 보세요.

00:42

[ 지금은 그만 ]
```

- 큰 글자와 높은 명도 대비를 사용한다.
- 화면을 꺼지지 않게 유지하는 기능은 사용자가 직접 루틴을 시작한 60초 동안에만 적용한다.
- 이미지, 영상, 자세 체크, 운동 횟수 카운트는 제공하지 않는다.
- `지금은 그만` 선택 시 즉시 홈으로 돌아가며 부정적 피드백을 표시하지 않는다.

#### A-08. Android 완료 상태

텍스트 구성:

```text
충분해요.
다시 당신의 흐름으로 돌아가요.
```

- 노출: 4초
- 이후 홈 화면으로 이동한다.
- 다음 알림은 완료 시점부터 60분 후로 재설정한다.

#### A-09. 홈 화면 위젯

**목적:** 앱을 열지 않고도 상태를 확인·제어하게 한다.

작은 위젯:

```text
TeuM
다음 틈까지 24분
[ 지금 1분 ]
```

큰 위젯:

```text
틈새움
다음 틈까지 24분
물 한 모금, 1분의 움직임

[ 지금 1분 ]  [ 잠시 멈춤 ]
```

- 위젯은 완료 횟수나 건강 수치를 표시하지 않는다.
- `지금 1분`을 누르면 루틴 화면을 연다.
- `잠시 멈춤`을 누르면 30분 일시정지를 즉시 적용하고, 가벼운 토스트로만 알린다.

#### A-10. Android 설정

섹션 구성:

```text
업무 시간
시작 [09:00]   종료 [18:00]
반복 요일 [월 화 수 목 금]

알림 방식
알림 [켜짐]
방식 [무음 / 진동]
화면 상단에 잠깐 표시 [켜짐]

개인정보
틈새움은 건강·행동 데이터를 수집하거나 외부로 전송하지 않습니다.
```

- `화면 상단에 잠깐 표시`를 끄면 헤드업 없이 알림 영역에만 남긴다.
- 알림 권한이 꺼져 있으면 시스템 설정으로 가는 버튼을 표시한다.
- 모든 설정은 기기 내에만 저장한다.

## 7. 상태 및 알림 흐름

```text
[업무 시간 시작]
        ↓
[60분 대기]
        ↓
[휴식 제안]
   ├─ 지금 1분 → [1분 텍스트 루틴] → [완료] → 완료 시점부터 60분 대기
   ├─ 5분 뒤   → [5분 대기] → [휴식 제안 1회]
   └─ 넘기기   → [다음 정규 알림까지 대기]
```

추가 규칙:

- `5분 뒤` 재알림에도 반응이 없으면 더 이상 재촉하지 않는다.
- 업무 종료 시각을 지나면 모든 보류·재알림을 취소한다.
- 일시정지 중에는 루틴 화면 및 예약 알림을 열지 않는다.
- 사용자가 수동으로 `지금 1분`을 완료하면 이전에 예정된 알림은 새 완료 시점 기준으로 다시 계산한다.

## 8. 접근성 및 표현 기준

- 모든 핵심 버튼은 손가락으로 누르기 쉬운 충분한 크기로 제공한다.
- 색만으로 상태를 구분하지 않고 텍스트를 함께 표시한다.
- 알림과 버튼의 명칭을 일관되게 유지한다: `지금 1분`, `5분 뒤`, `이번엔 넘기기`.
- 진동을 켠 사용자는 시각 정보도 함께 받아야 하며, 진동만으로 의미를 전달하지 않는다.
- 타이머는 초조함을 유발하는 경고색·깜빡임·큰 소리를 사용하지 않는다.
- 사용자가 시스템의 글자 크기를 키워도 핵심 문장과 3개 버튼이 잘리지 않게 설계한다.

## 9. MVP 출시 기준

다음 조건을 만족하면 MVP를 출시할 수 있다.

- PC와 Android에서 업무 시간 안에 60분 주기 알림이 작동한다.
- Android에서 무음/진동을 선택할 수 있다.
- PC 알림 배너는 12초, Android 헤드업은 약 8초 노출 후 비차단 상태로 전환된다.
- 모든 알림에서 `지금 1분`, `5분 뒤`, `이번엔 넘기기`를 사용할 수 있다.
- 루틴 화면은 60초 동안 텍스트만 안내한다.
- 완료 시점부터 다음 60분을 계산한다.
- 앱은 사용자의 건강·행동 데이터를 수집하거나 서버로 전송하지 않는다.
- 사용자는 언제든 알림을 일시정지하거나 끌 수 있다.

## 10. 이후 검토할 항목 (MVP 제외)

- PC와 Android 간 설정 동기화 여부
- 회사용 배포 및 조직 공통 업무 시간 설정
- 사용자가 선택할 수 있는 다른 휴식 문구
- 화면 공유·회의 상태를 더 정확히 감지하는 범위와 권한
- 건강 관련 안내 문구의 전문가 검토

이 항목들은 기본 경험이 안정화된 뒤 검토하며, 데이터 수집을 전제로 하지 않는다.
