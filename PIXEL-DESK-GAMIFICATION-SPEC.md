# 틈새움 픽셀 데스크 기획서

> 문서 버전: 1.0
> 대상: Android 우선, PC 동일 콘셉트 확장
> 적용 목표: 1.0.4 이후
> 핵심 문장: **사용자가 몸을 챙길 때, 작은 업무 공간이 조금 밝아진다.**

## 1. 기획 결론

픽셀 데스크는 별도의 게임 화면이 아니다. 기존 알림을 완료했을 때 홈 화면의 작은 장면이 조용히 변하는 **수동형 보상 레이어**다.

알림의 핵심 흐름은 바꾸지 않는다.

```text
알림 → 1분의 틈 → 네, 챙겼어요 → 홈으로 복귀
```

완료 후 추가 보상 버튼, 점수, 레벨, 랭킹, 연속 기록, 실패 상태는 만들지 않는다.

### 사용자별 기대값

| 사용자 | 경험 |
|---|---|
| 가볍게 쓰는 사람 | 기존 완료 버튼만 누르면 장면이 자동으로 변한다. 픽셀 화면을 따로 열 필요가 없다. |
| 기록을 좋아하는 사람 | 기록 화면에서 날짜별 횟수와 해당 날의 장면 상태를 함께 볼 수 있다. |
| 건강 목적 사용자 | 점수 대신 오늘 몸을 챙긴 횟수와 다음 알림을 확인한다. |

## 2. 장면의 역할

픽셀 데스크는 다음 세 가지를 전달해야 한다.

1. **행동 결과:** 방금 한 번의 틈을 챙겼다는 사실
2. **누적 감각:** 오늘 몇 번 챙겼는지 한눈에 보이는 변화
3. **낮은 압박:** 건너뛰어도 장면이 어두워지거나 초기화되지 않는다는 안정감

픽셀 데스크가 건강 수치를 대신하면 안 된다. 물의 양, 칼로리, 자세 정확도, 신체 점수를 표시하지 않는다.

## 3. 장면 구조

### 3.1 논리 그리드

모든 장면은 16×16 논리 픽셀 단위를 기준으로 제작한다.

| 항목 | 기준 |
|---|---:|
| 논리 캔버스 | 12×8 타일 = 192×128 logical px |
| Android 표시 배율 | 1.5~2.0배, 정수에 가까운 배율 우선 |
| PC 표시 배율 | 2~3배 |
| 외곽 여백 | 논리 타일 1개 이상 |
| 안티앨리어싱 | 금지 |
| 색상 수 | 한 장면 최대 8색 |
| 애니메이션 | 3프레임 이내, 1초 이하 |

### 3.2 장면 레이어

뒤에서 앞으로 다음 순서로 합성한다.

1. **벽/배경:** `canvasSoft` 또는 `platinum`
2. **카본 창틀:** `carbon`
3. **책상:** `chromeIndigo`, `periwinkle`
4. **모니터·물컵·식물:** 상태에 따라 변경
5. **작은 신호등:** `amber` 또는 `signal`
6. **하이라이트:** `highlight`, 최대 2픽셀 두께

Nintendo 캐릭터나 컨트롤러를 넣지 않는다. 기존 디자인의 하드웨어 베젤과 사무실 오브젝트만 사용한다.

## 4. Android 현재 화면 구성 기준

현재 Android 소스의 실제 레이아웃 토큰을 기준으로 한다. `SafeAreaView`·상태 표시줄·52dp 커맨드 바 다음에 화면이 시작되고, 모든 `ScrollView`는 좌우 14dp 패딩과 14dp 카드 간격을 사용한다. 알림 권한 안내 패널이 추가되더라도 아래 순서 자체는 바꾸지 않는다. 화면 높이는 카드 내용과 글꼴에 따라 달라지므로 고정 y좌표 대신 **현재 카드 안의 상대 위치**를 기준으로 잡는다.

### 4.1 홈 — Hero 카드 안쪽

현재 순서: `NOTICE(조건부) → Hero → TODAY'S RHYTHM → 나의 틈 기록(기록 모드일 때) → CONTROL PANEL`.

- 픽셀 데스크는 Hero 카드의 마지막 요소로 추가한다. `Eyebrow → 제목/남은 시간 → 설명 → 진행 미터` 아래에 `PixelDeskRow`를 둔다.
- `PixelDeskRow`는 우측 정렬하며 장면은 96×64dp, 바깥 여백은 8dp로 둔다. 절대 위치로 겹치게 배치하지 않는다.
- Hero 제목·남은 시간·설명은 장면 영역을 침범하지 않도록 최대 폭을 제한한다. 좁은 화면에서는 장면을 80×54dp로 줄이고 텍스트를 우선한다.
- `PAUSED`, `PAUSE SIGNAL`, 업무 외 시간 상태에도 같은 자리를 유지하되 장면을 정지 상태로 보여 준다. 상태마다 카드 높이가 크게 바뀌지 않게 한다.
- 장면은 버튼이 아니며 탭·게임 화면 이동을 제공하지 않는다. 접근성은 카드의 요약 문장으로 읽는다.

### 4.2 홈 — TODAY'S RHYTHM 패널

현재 세 개의 `InfoRow`를 유지하고, 패널 탭 바로 아래에 `오늘의 틈` 슬롯 줄 하나만 추가한다.

```text
오늘의 틈  [■][■][■][□][□][□][□][□]
           1  2  3  4  5  6  7  8
```

- 슬롯 한 칸: 18×18dp
- 슬롯 간격: 6dp
- 완료 슬롯: `amber`
- 예정·미완료 슬롯: `carbon` 안쪽 + `hairline` 외곽
- 넘긴 슬롯: 색을 바꾸지 않고 비워 둔다.
- 8개를 넘는 경우 슬롯은 모두 채운 상태를 유지하고, 텍스트에만 `오늘 9번 챙겼어요`를 표시한다.
- 슬롯 줄은 `InfoRow`보다 먼저 보여 주며, 기존 업무 시간·알람 방식·현재 상태의 정보 우선순위를 바꾸지 않는다.

### 4.3 홈 — 나의 틈 기록 / CONTROL PANEL

- `나의 틈 기록` 패널에는 큰 픽셀 장면을 추가하지 않는다. 현재처럼 한 줄 요약과 `기록 보기` 버튼만 둬야 가벼운 사용자에게 부담이 없다.
- `CONTROL PANEL`, 알림 권한 안내, 토스트, 일시정지 바텀시트에도 픽셀 장면을 넣지 않는다. 조작·권한·오류 피드백과 장식이 섞이지 않게 한다.

### 4.4 1분의 틈 화면

알림에서 열린 화면은 행동에 집중해야 하므로 픽셀 데스크를 작게만 사용한다.

- 현재 `Plate`의 중앙 세로 흐름을 유지한다: 제목/타이머(또는 `어땠나요?`) → 안내 문장 → 질문 → `PixelDeskMini` → 버튼.
- 위치: 질문 문장 바로 아래, `actions` 컨테이너 바로 위
- 크기: 88×56dp, 중앙 정렬
- 타이머와 버튼 사이의 넓은 공간을 차지하지 않도록 한다.
- `네, 챙겼어요`를 누르면 물컵의 한 칸이 차거나 창문에 한 칸의 빛이 켜지는 0.8초 변화를 재생한다.
- 별도의 보상 모달을 열지 않고 곧바로 홈으로 돌아간다.
- `이번엔 넘길게요`를 누르면 장면은 그대로 유지한다.

### 4.5 기록 화면

달력은 사실 확인용, 픽셀 데스크는 감정 확인용으로 분리한다.

- 달력 셀에는 픽셀을 넣지 않는다. 현재 달력은 날짜와 횟수 확인에 집중한다.
- 선택 날짜 패널의 탭 바로 아래에 `SelectedDaySummary`를 추가한다.
- 왼쪽에는 현재 날짜·횟수 요약, 오른쪽에는 104×72dp 픽셀 장면을 둔다.
- 그 아래 기존 시간대별 `recordRow`를 그대로 둔다. 기록이 없는 날에도 오른쪽 장면은 기본 책상 상태를 유지한다.
- 기록이 없는 날에도 어둡거나 깨진 장면을 보여 주지 않는다. 기본 책상 장면을 유지한다.

### 4.6 설정·온보딩

- 현재 `SettingsScreen`과 `Onboarding`에는 픽셀 데스크를 넣지 않는다. 알람 방식, 권한, 기록 모드, 언어를 설정하는 화면이므로 장면 변화는 홈·기록·1분 화면에서만 보여 준다.

## 5. PC 화면 위치 — Android와 동일한 정보 우선순위

### 5.1 메뉴바 팝오버

- 팝오버 기준 크기: 320×240px
- 장면 위치: 우측 상단 112×80px
- 왼쪽 영역: 다음 알림 시각과 `지금 1분` 버튼
- 하단: 오늘의 8슬롯
- 픽셀 장면은 팝오버를 열었을 때만 렌더링한다. 상주 애니메이션은 사용하지 않는다.

### 5.2 PC 1분 안내 창

- 창 기준 크기: 420×300px
- 상단: `1분의 틈` 라벨
- 중앙: 128×88px 픽셀 데스크
- 하단: 안내 문장과 완료·넘기기 선택
- 화면 공유·전체 화면 중에는 픽셀 창과 알림을 자동으로 띄우지 않는다.

## 6. 상태와 변화 규칙

### 6.1 상태별 표현

| 상태 | 장면 표현 | 문구 |
|---|---|---|
| 기록 없음 | 모니터 대기 화면, 작은 스탠드 | `오늘의 첫 틈을 기다리고 있어요.` |
| 1~2회 완료 | 물컵 또는 모니터에 작은 amber 픽셀 | `오늘 2번 챙겼어요.` |
| 3~5회 완료 | 창문 빛 1~2칸, 식물 한 잎 | `오늘의 자리도 조금 밝아졌어요.` |
| 6~8회 완료 | 스탠드·창문·식물 모두 표시 | `오늘 여러 번 몸을 챙겼어요.` |
| 일시정지 | 장면은 유지, 신호등만 정지 | `잠시 쉬어 가는 중이에요.` |
| 넘김 | 장면 유지 | 별도 부정 문구 없음 |

### 6.2 하루 리셋

- 자정에 장면을 완전히 초기화하지 않는다.
- 다음 날 첫 실행 시 기본 책상으로 전환하되, 전날 장면은 기록 화면에서 확인할 수 있다.
- 연속 일수나 끊김을 표시하지 않는다.
- 일주일 장면은 별도 보상으로 만들지 않고 기록 화면의 주간 요약에서만 선택적으로 보여 준다.

### 6.3 해금 규칙

해금은 점수 보상이 아니라 시각적 상태 변화다.

| 누적 완료 | 변화 |
|---:|---|
| 1회 | 물컵 표시 |
| 3회 | 창문 빛 1칸 |
| 5회 | 식물 표시 |
| 8회 | 스탠드 켜짐 |

하루가 끝나면 다음 날 기본 상태로 돌아가며, 기록 데이터가 있는 날에는 과거 장면을 다시 계산할 수 있다.

## 7. HCI·접근성 기준

- 픽셀 장면은 정보를 독점하지 않는 장식/요약 요소로 표시한다.
- 접근성 라벨 예시: `오늘 3번의 틈을 챙겼어요. 책상 장면에 창문 빛이 켜져 있어요.`
- 색만으로 완료 여부를 구분하지 않는다. 슬롯 수와 문장을 함께 제공한다.
- `움직임 줄이기` 설정이 켜지면 모든 프레임 애니메이션을 정지 화면으로 대체한다.
- 장면의 깜빡임 주기는 1초보다 짧지 않으며, 알림 진동과 동기화하지 않는다.
- 한국어 본문은 기존 Pretendard/시스템 글꼴을 유지한다. 픽셀 폰트는 작은 영문 라벨에만 제한한다.

## 8. 구현 단위

### Android

```text
PixelGlyph          8×8/16×16 색상 배열 렌더러
PixelScene          desk/window/glass/plant/lamp 레이어 조합
PixelSlotMeter      오늘의 8칸
PixelSceneState     doneCount와 pause 상태를 장면 상태로 변환
```

- React Native `View` 사각형을 사용해 코드로 만든 기본 오브젝트를 먼저 구현한다.
- 외부 PNG를 사용할 경우 논리 캔버스에 맞춰 nearest-neighbor로만 확대한다.
- 모든 외부 자산은 앱의 `android/assets`로 바로 복사하지 않고, 검토 후 필요한 파일만 복사한다.
- 사용자의 기록 모드가 꺼져 있으면 `doneCount`는 홈 픽셀 데스크에 사용하지 않는다.

### PC

- CSS `image-rendering: pixelated` 또는 Canvas의 정수 배율 사용
- 메뉴바 팝오버가 열릴 때만 장면 계산
- 운영체제별 애니메이션 감소 설정을 감지하고 정지 장면 제공

## 9. 오픈소스 자산 사용 원칙

현재 저장소의 `opensource/` 아래에 후보 저장소를 원본 그대로 보관한다. 원본 저장소 전체를 앱 번들에 포함하지 않는다.

| 저장소 | 라이선스 | 활용 범위 | 주의사항 |
|---|---|---|---|
| `halfmage/pixelarticons` | MIT | 알람·벨·달력·물컵 등 보조 SVG 아이콘 | 필요한 SVG만 복사하고 MIT 고지를 유지한다. |
| `pixel-agents-hq/pixel-agents` | MIT(프로젝트), 사무용 자산 공개 | 책상·PC·식물·벽 타일 레퍼런스 및 후보 자산 | 출처가 분리된 캐릭터·펫은 보관소에서 제거하고 사용하지 않는다. |
| `iwenzhou/kenney` | CC0-1.0 | 픽셀 팔레트·타일·소품 탐색 | 오래된 미러이므로 `LICENSE.md`를 함께 보관하고, 실제 사용 전 원 출처를 재확인한다. |

브랜드 장면의 핵심 오브젝트(물컵, 창문 빛, 식물, 스탠드)는 외부 자산을 그대로 조합하기보다 코드로 새로 그려 스타일을 통일한다. 외부 자산은 비교·참고·보조 아이콘에 우선 사용한다.

## 10. 출시 전 검증 기준

1. 알림 완료 후 추가 보상 화면 없이 2초 안에 홈으로 돌아온다.
2. 기록 모드가 꺼져 있으면 픽셀 데스크와 달력 기록이 노출되지 않는다.
3. `이번엔 넘길게요`를 눌러도 장면이 후퇴하지 않는다.
4. 360dp 폭에서 제목·남은 시간·주요 버튼이 픽셀 장면보다 우선 보인다.
5. TalkBack이 장면을 하나의 요약 문장으로 읽는다.
6. `움직임 줄이기` 설정에서 장면이 깜빡이지 않는다.
7. 원본 오픈소스 전체 디렉터리가 APK/AAB에 들어가지 않는다.
8. `THIRD_PARTY_NOTICES.md`에 실제 복사한 파일만 기록한다.
