# 틈새움 픽셀 데스크 기획서

> 문서 버전: 1.1
> 대상: Android 우선, PC 동일 콘셉트 확장
> 적용 목표: 1.0.4 이후 (아이템·꾸미기 확장은 1.1.x)
> 핵심 문장: **사용자가 몸을 챙길 때, 작은 업무 공간이 조금 밝아진다.**

## 1. 기획 결론

픽셀 데스크는 별도의 게임 화면이 아니다. 기존 알림을 완료했을 때 홈 화면의 작은 장면이 조용히 변하는 **수동형 보상 레이어**다.

알림의 핵심 흐름은 바꾸지 않는다.

```text
알림 → 1분의 틈 → 네, 챙겼어요 → 홈으로 복귀
```

완료 후 추가 보상 버튼, 점수, 레벨, 랭킹, 연속 기록, 실패 상태는 만들지 않는다.

### 확장을 전제로 한 구조 원칙

첫 버전(1.0.4)의 오브젝트 4개(물컵·창문 빛·식물·스탠드)도 **하드코딩하지 않고 아이템 카탈로그 + 슬롯 시스템** 위에 올린다. 이후 아이템 추가와 꾸미기(배치 선택)는 카탈로그에 항목을 더하고 꾸미기 UI만 여는 일이 되도록, 장면·데이터·저장 구조를 처음부터 분리한다.

| 계층 | 역할 | 확장 시 바뀌는 것 |
|---|---|---|
| 렌더러 (`PixelGlyph`) | 도트맵 배열을 사각형으로 그린다 | 바뀌지 않음 |
| 장면 (`PixelScene`) | 슬롯 좌표에 아이템 인스턴스를 합성한다 | 바뀌지 않음 |
| 카탈로그 (`ItemCatalog`) | 아이템 정의(도트맵·크기·허용 슬롯·획득 조건) | **여기에만 추가** |
| 상태 (`DeskState`) | 획득한 아이템·배치 선택 | 스키마 버전으로 마이그레이션 |

### 사용자별 기대값

| 사용자 | 경험 |
|---|---|
| 가볍게 쓰는 사람 | 기존 완료 버튼만 누르면 장면이 자동으로 변한다. 픽셀 화면을 따로 열 필요가 없다. |
| 기록을 좋아하는 사람 | 기록 화면에서 날짜별 횟수와 해당 날의 장면 상태를 함께 볼 수 있다. |
| 건강 목적 사용자 | 점수 대신 오늘 몸을 챙긴 횟수와 다음 알림을 확인한다. |

## 2. 장면의 역할

픽셀 데스크는 다음 세 가지를 전달해야 한다.

1. **행동 결과:** 방금 한 번의 틈을 챙겼다는 사실
2. **누적 감각:** 오늘 몇 번 챙겼는지 한눈에 보이는 변화
3. **낮은 압박:** 건너뛰어도 장면이 어두워지거나 초기화되지 않는다는 안정감

픽셀 데스크가 건강 수치를 대신하면 안 된다. 물의 양, 칼로리, 자세 정확도, 신체 점수를 표시하지 않는다.

## 3. 장면 구조

### 3.1 논리 그리드

모든 장면은 16×16 논리 픽셀 단위를 기준으로 제작한다.

| 항목 | 기준 |
|---|---:|
| 논리 캔버스 | 16×10 타일 = 256×160 logical px (슬롯 확장을 수용하는 크기) |
| Android 홈 표시 | 128×80dp — 좁은 화면 112×70dp. 장면 탭 → 픽셀 데스크 화면 |
| Android 1분 화면 표시 | 120×75dp (탭 없음 — 행동 집중) |
| Android 기록 표시 | 128×80dp. 장면 탭 → 픽셀 데스크 화면 |
| Android 픽셀 데스크 화면 | 콘텐츠 전체 폭 (약 300dp+) — 가장 큰 뷰 |
| PC 표시 배율 | 팝오버 128×80px, 1분 창 192×120px |
| 외곽 여백 | 논리 타일 1개 이상 |
| 안티앨리어싱 | 금지 |
| 색상 수 | 한 장면 최대 8색 |
| 애니메이션 | 3프레임 이내, 1초 이하 |

표시 배율이 정수가 아니어도 `View` 사각형 렌더링에는 필터링 흐림이 없다. 다만 인접 사각형 사이 헤어라인 틈을 막기 위해, 렌더러는 사각형 경계를 물리 픽셀에 스냅하거나 0.5px 겹침을 사용한다.

### 3.2 장면 레이어

뒤에서 앞으로 다음 순서로 합성한다.

1. **벽/배경:** `canvasSoft` 또는 `platinum`
2. **카본 창틀:** `carbon`
3. **책상:** `chromeIndigo`, `periwinkle`
4. **아이템 레이어:** 슬롯에 배치된 아이템 인스턴스 (모니터·물컵·식물 등)
5. **작은 신호등:** `amber` 또는 `signal`
6. **하이라이트:** `highlight`, 최대 2픽셀 두께

Nintendo 캐릭터나 컨트롤러를 넣지 않는다. 기존 디자인의 하드웨어 베젤과 사무실 오브젝트만 사용한다.

### 3.3 슬롯 시스템 (꾸미기 확장의 기반)

장면 위 아이템 위치는 자유 좌표가 아니라 **고정 슬롯**으로 제한한다. 슬롯은 배치 충돌·겹침·잘림 문제를 구조적으로 없애고, 꾸미기 UI를 "슬롯마다 아이템 고르기"라는 단순한 형태로 유지해 준다.

장면은 벽 → 책상 → 바닥의 3개 존으로 나뉘고, 슬롯은 존에 소속된다. 16×10 타일 기준 슬롯 지도:

```text
행 0-3   [시계]      [창문 4×3]        [선반 a][선반 b]   [액자]
행 4-6              ── 책상 상판 ──
         [데스크L]  [모니터(고정)]  [데스크R]  [스탠드]
행 7     [앞줄 소품: 메모·키보드 옆]
행 8-9   [바닥L: 큰 화분·박스]                    [바닥R]
```

| 슬롯 id | 존 | 허용 크기(논리 px) | 기본 아이템 (1.0.4) |
|---|---|---|---|
| `wall-window` | 벽 | 고정 창틀 내부 64×48 | 창문 빛 |
| `wall-shelf-a` | 벽 | 14×10 | 비움 (1.1.x 개방) |
| `wall-shelf-b` | 벽 | 14×10 | 비움 (1.1.x 개방) |
| `wall-frame` | 벽 | 16×20 | 비움 (1.1.x 개방) |
| `wall-clock` | 벽 | 12×12 | 비움 (1.1.x 개방) |
| `desk-left` | 책상 | 16×16 | 물컵 |
| `desk-center` | 책상 | 48×32 | 모니터 (고정, 교체 불가) |
| `desk-right` | 책상 | 16×16 | 식물 |
| `desk-lamp` | 책상 | 12×24 | 스탠드 |
| `desk-front` | 책상 | 24×8 | 비움 (1.1.x 개방) |
| `floor-left` | 바닥 | 20×28 | 비움 (1.1.x 개방) |
| `floor-right` | 바닥 | 20×28 | 비움 (1.1.x 개방) |

- 슬롯 좌표·크기는 `PixelScene`이 소유한다. 아이템은 자기 위치를 모른다.
- 1.0.4에서는 기본 아이템 5개 자리만 쓰이고 해금 상태만 바뀐다. 나머지 7개 슬롯은 **비워 둔 채 출시**한다 — 코드가 아니라 카탈로그가 비어 있는 것이므로, 확장 시 데이터만 추가하면 된다.
- 새 슬롯 추가는 장면 개편이므로 마이너 버전에서만 한다. 아이템 추가는 언제든 가능하다.

### 3.3.1 조판 규칙 — 슬롯이 늘어도 이뻐 보이게

슬롯이 많다고 다 채우면 장면이 시장통이 된다. 다음 규칙으로 밀도와 위계를 지킨다.

| 규칙 | 기준 |
|---|---|
| 여백 우선 | 인접 슬롯 사이 최소 1타일(16px)의 빈 공간. 슬롯 지도 자체가 이 간격을 보장한다. |
| 빈 슬롯은 투명 | 미배치 슬롯에 플레이스홀더·점선·자물쇠를 그리지 않는다. 비어 있으면 그냥 벽과 책상이다. |
| 밀도 상한 | 동시에 그리는 아이템은 모니터 포함 **최대 8개**. 초과 배치는 꾸미기 UI에서 막는다. |
| 색 위계 | `amber`·`signal` 포인트 색은 한 장면에 **3곳 이하**. 벽·바닥 존 아이템은 저채도 팔레트(`canvasSoft`/`periwinkle`/`mutedIndigo`/`carbon`)만 사용해 책상 존이 항상 주인공이 되게 한다. |
| 시선 흐름 | 밝기 중심은 창문 빛 → 모니터 → 스탠드의 대각선을 유지한다. 이 축을 가리는 아이템은 만들지 않는다. |
| 지평선 고정 | 책상 상판(행 6)과 바닥선(행 8)은 어떤 배치에서도 끊기지 않는 수평선으로 남긴다. |

이 규칙은 꾸미기 UI의 제약이기도 하다: 사용자가 어떤 조합을 골라도 장면이 규칙을 벗어날 수 없게, 제약을 카탈로그(존별 팔레트)와 슬롯 지도(간격)에 미리 심는다.

### 3.4 아이템 카탈로그 규격

모든 아이템은 아래 스키마로 정의한다. 첫 버전의 4개 오브젝트도 동일하다.

```text
id            안정적인 문자열 id (예: glass-basic). 한 번 배포하면 바꾸지 않는다.
name          표시 이름 (i18n 키)
slots         배치 가능한 슬롯 id 목록
size          논리 픽셀 크기 (슬롯 허용 크기 이내)
frames        도트맵 배열 1~3프레임 (브랜드 팔레트 토큰만 사용)
states        상태별 도트맵 변형 (예: 물컵 빈/반/가득) — 선택
acquire       획득 조건 (아래 6.3) — `default`는 처음부터 소유
addedIn       추가된 앱 버전 (마이그레이션·기록 재계산용)
```

- 도트맵은 색상 토큰의 2차원 배열 데이터다. PNG를 쓰지 않고 코드(데이터)로 관리한다.
- 아이템 도트 규격: 색 최대 8개(팔레트 토큰만), 애니메이션 최대 3프레임·1초 이하, 안티앨리어싱 금지, 외곽 1px `carbon` 윤곽 권장.
- 카탈로그는 앱에 내장한다. 서버 다운로드·원격 구성은 하지 않는다.

## 4. Android 현재 화면 구성 기준

현재 Android 소스의 실제 레이아웃 토큰을 기준으로 한다. `SafeAreaView`·상태 표시줄·52dp 커맨드 바 다음에 화면이 시작되고, 모든 `ScrollView`는 좌우 14dp 패딩과 14dp 카드 간격을 사용한다. 알림 권한 안내 패널이 추가되더라도 아래 순서 자체는 바꾸지 않는다. 화면 높이는 카드 내용과 글꼴에 따라 달라지므로 고정 y좌표 대신 **현재 카드 안의 상대 위치**를 기준으로 잡는다.

### 4.1 홈 — Hero 카드 안쪽

현재 순서: `NOTICE(조건부) → Hero → TODAY'S RHYTHM → 나의 틈 기록(기록 모드일 때) → CONTROL PANEL`.

- 픽셀 데스크는 Hero 카드의 마지막 요소로 추가한다. `Eyebrow → 제목/남은 시간 → 설명 → 진행 미터` 아래에 `PixelDeskRow`를 둔다.
- `PixelDeskRow`는 우측 정렬하며 장면은 96×60dp, 바깥 여백은 8dp로 둔다. 절대 위치로 겹치게 배치하지 않는다.
- Hero 제목·남은 시간·설명은 장면 영역을 침범하지 않도록 최대 폭을 제한한다. 좁은 화면에서는 장면을 80×50dp로 줄이고 텍스트를 우선한다.
- `PAUSED`, `PAUSE SIGNAL`, 업무 외 시간 상태에도 같은 자리를 유지하되 장면을 정지 상태로 보여 준다. 상태마다 카드 높이가 크게 바뀌지 않게 한다.
- 장면은 버튼이 아니며 탭·게임 화면 이동을 제공하지 않는다. 접근성은 카드의 요약 문장으로 읽는다.

### 4.2 홈 — TODAY'S RHYTHM 패널

현재 세 개의 `InfoRow`를 유지하고, 패널 탭 바로 아래에 `오늘의 틈` 슬롯 줄 하나만 추가한다.

```text
오늘의 틈  [■][■][■][□][□][□][□][□]
           1  2  3  4  5  6  7  8
```

- 슬롯 한 칸: 18×18dp
- 슬롯 간격: 6dp
- 완료 슬롯: `amber`
- 예정·미완료 슬롯: `carbon` 안쪽 + `hairline` 외곽
- 넘긴 슬롯: 색을 바꾸지 않고 비워 둔다.
- 8개를 넘는 경우 슬롯은 모두 채운 상태를 유지하고, 텍스트에만 `오늘 9번 챙겼어요`를 표시한다.
- 슬롯 줄은 `InfoRow`보다 먼저 보여 주며, 기존 업무 시간·알람 방식·현재 상태의 정보 우선순위를 바꾸지 않는다.

### 4.3 홈 — 나의 틈 기록 / CONTROL PANEL

- `나의 틈 기록` 패널에는 큰 픽셀 장면을 추가하지 않는다. 현재처럼 한 줄 요약과 `기록 보기` 버튼만 둬야 가벼운 사용자에게 부담이 없다.
- `CONTROL PANEL`, 알림 권한 안내, 토스트, 일시정지 바텀시트에도 픽셀 장면을 넣지 않는다. 조작·권한·오류 피드백과 장식이 섞이지 않게 한다.

### 4.4 1분의 틈 화면

알림에서 열린 화면은 행동에 집중해야 하므로 픽셀 데스크를 작게만 사용한다.

- 현재 `Plate`의 중앙 세로 흐름을 유지한다: 제목/타이머(또는 `어땠나요?`) → 안내 문장 → 질문 → `PixelDeskMini` → 버튼.
- 위치: 질문 문장 바로 아래, `actions` 컨테이너 바로 위
- 크기: 96×60dp, 중앙 정렬
- 타이머와 버튼 사이의 넓은 공간을 차지하지 않도록 한다.
- `네, 챙겼어요`를 누르면 물컵의 한 칸이 차거나 창문에 한 칸의 빛이 켜지는 0.8초 변화를 재생한다.
- 별도의 보상 모달을 열지 않고 곧바로 홈으로 돌아간다.
- `이번엔 넘길게요`를 누르면 장면은 그대로 유지한다.

### 4.5 기록 화면

달력은 사실 확인용, 픽셀 데스크는 감정 확인용으로 분리한다.

- 달력 셀에는 픽셀을 넣지 않는다. 현재 달력은 날짜와 횟수 확인에 집중한다.
- 선택 날짜 패널의 탭 바로 아래에 `SelectedDaySummary`를 추가한다.
- 왼쪽에는 현재 날짜·횟수 요약, 오른쪽에는 128×80dp 픽셀 장면을 둔다.
- 그 아래 기존 시간대별 `recordRow`를 그대로 둔다. 기록이 없는 날에도 오른쪽 장면은 기본 책상 상태를 유지한다.
- 기록이 없는 날에도 어둡거나 깨진 장면을 보여 주지 않는다. 기본 책상 장면을 유지한다.

### 4.7 픽셀 데스크 화면 (전용, 1.0.4 포함)

홈·기록의 작은 장면을 누르면 열리는 **가장 큰 뷰이자 꾸미기 공간**이다. 집 꾸미기류 앱처럼 장면 자체가 화면의 주인공이 된다.

화면 구성 (위에서 아래로):

```text
[헤더: 틈새움 ── 닫기]

┌ 나의 책상 ──────────────────────┐
│  (전체 폭 픽셀 장면, ~300dp)      │
│  오늘 3번 챙겼어요.               │
└──────────────────────────────┘

┌ 내 아이템 ──────────────────────┐
│ [머그][책][액자][시계][화분] …    │
│ 틈을 챙기면 새 아이템이 도착해요.  │
└──────────────────────────────┘
```

- **장면 = 꾸미기 도구다.** 큰 장면 위의 슬롯 영역을 직접 누르면 그 슬롯에 넣을 수 있는 소장 아이템 시트가 열린다 (`비우기` 포함). 고르면 즉시 장면에 반영·저장된다.
- 드래그·회전·크기 조절은 없다. 슬롯 탭 → 아이템 선택이 전부다.
- **내 아이템(아이템란):** 소장 아이템을 도트 미리보기 그리드로 보여 준다. 미소장 아이템은 실루엣·자물쇠로 노출하지 않는다(§6.4). 그리드 아래 한 줄로만 다음 도착을 암시한다: `틈을 챙기면 새 아이템이 도착해요.`
- 아이템을 누르면 그 아이템이 들어갈 수 있는 슬롯으로 바로 배치한다(호환 슬롯이 하나일 때). 여러 슬롯이 가능하면 슬롯 선택 시트를 띄운다.
- `가볍게 사용`(기록 꺼짐)에서는 기본 장면과 기본 아이템만 보이고, 새 아이템 도착 문구 대신 `기록하며 사용을 켜면 책상이 자라나요.`를 보여 준다.
- 점수·진행률·다음 해금까지 N회 같은 압박 수치는 표시하지 않는다.

#### 1.0.4 마일스톤 아이템 (누적 챙김 기준)

| 아이템 | 슬롯 | 도착 시점(누적) |
|---|---|---:|
| 머그컵 | desk-left | 3회 |
| 선반 책 | wall-shelf-a/b | 7회 |
| 벽 액자 | wall-frame | 12회 |
| 벽시계 | wall-clock | 20회 |
| 큰 화분 | floor-left/right | 30회 |

- 도착은 조용히 한다: 축하 모달 없이, 다음에 데스크 화면을 열면 아이템란에 놓여 있다.
- 누적 챙김은 `teum.desk.v1`의 `cumulativeDone` 카운터로 관리한다. 기록 삭제와 독립이며, `기록하며 사용`에서 챙길 때만 오른다.

### 4.6 설정·온보딩

- 현재 `SettingsScreen`과 `Onboarding`에는 픽셀 데스크를 넣지 않는다. 알람 방식, 권한, 기록 모드, 언어를 설정하는 화면이므로 장면 변화는 홈·기록·1분 화면에서만 보여 준다.

## 5. PC 화면 위치 — Android와 동일한 정보 우선순위

### 5.1 메뉴바 팝오버

- 팝오버 기준 크기: 320×240px
- 장면 위치: 우측 상단 128×80px
- 왼쪽 영역: 다음 알림 시각과 `지금 1분` 버튼
- 하단: 오늘의 8슬롯
- 픽셀 장면은 팝오버를 열었을 때만 렌더링한다. 상주 애니메이션은 사용하지 않는다.

### 5.2 PC 1분 안내 창

- 창 기준 크기: 420×300px
- 상단: `1분의 틈` 라벨
- 중앙: 192×120px 픽셀 데스크
- 하단: 안내 문장과 완료·넘기기 선택
- 화면 공유·전체 화면 중에는 픽셀 창과 알림을 자동으로 띄우지 않는다.

## 6. 상태와 변화 규칙

### 6.1 상태별 표현

| 상태 | 장면 표현 | 문구 |
|---|---|---|
| 기록 없음 | 모니터 대기 화면, 작은 스탠드 | `오늘의 첫 틈을 기다리고 있어요.` |
| 1~2회 완료 | 물컵 또는 모니터에 작은 amber 픽셀 | `오늘 2번 챙겼어요.` |
| 3~5회 완료 | 창문 빛 1~2칸, 식물 한 잎 | `오늘의 자리도 조금 밝아졌어요.` |
| 6~8회 완료 | 스탠드·창문·식물 모두 표시 | `오늘 여러 번 몸을 챙겼어요.` |
| 일시정지 | 장면은 유지, 신호등만 정지 | `잠시 쉬어 가는 중이에요.` |
| 넘김 | 장면 유지 | 별도 부정 문구 없음 |

### 6.2 하루 리셋

- 자정에 장면을 완전히 초기화하지 않는다.
- 다음 날 첫 실행 시 기본 책상으로 전환하되, 전날 장면은 기록 화면에서 확인할 수 있다.
- 연속 일수나 끊김을 표시하지 않는다.
- 일주일 장면은 별도 보상으로 만들지 않고 기록 화면의 주간 요약에서만 선택적으로 보여 준다.

### 6.3 해금 규칙 — 오늘의 상태 변화 (자동)

해금은 점수 보상이 아니라 시각적 상태 변화다. 카탈로그의 `acquire` 조건 중 `daily` 유형에 해당한다.

| 누적 완료 | 변화 |
|---:|---|
| 1회 | 물컵 표시 |
| 3회 | 창문 빛 1칸 |
| 5회 | 식물 표시 |
| 8회 | 스탠드 켜짐 |

하루가 끝나면 다음 날 기본 상태로 돌아가며, 기록 데이터가 있는 날에는 과거 장면을 다시 계산할 수 있다.

### 6.4 소장과 꾸미기 (1.1.x 확장)

하루 단위로 리셋되는 6.3의 상태 변화와 별개로, **소장 아이템**은 누적되어 사라지지 않는다.

- 획득(`acquire`) 유형:
  - `default` — 처음부터 소유 (기본 물컵·식물·스탠드 등)
  - `daily` — 오늘의 완료 횟수에 따른 당일 상태 변화 (소장 아님)
  - `milestone` — 누적 완료 N회 도달 시 소장 (예: 새 물컵 도색, 다른 화분). 도달 사실만 조용히 알리고 축하 모달·연출은 띄우지 않는다.
  - `seasonal` — 특정 기간에 앱을 사용한 날 소장 (예: 12월의 작은 트리). 기간이 지나도 소장품은 회수하지 않는다.
- 꾸미기 UI: 기록 화면에서 진입하는 `내 책상 꾸미기` 한 화면. 슬롯을 누르면 그 슬롯에 넣을 수 있는 소장 아이템 목록이 뜨고, 고르면 즉시 반영된다. 배치는 슬롯 단위 선택뿐이며 드래그·회전·크기 조절은 제공하지 않는다.
- 꾸미기에서 지켜야 할 것:
  - 재화·상점·구매·광고 보상은 만들지 않는다. 아이템은 오직 몸을 챙긴 날들로만 모인다.
  - 아이템을 잃거나 후퇴하는 규칙은 만들지 않는다. 넘기거나 쉬어도 소장품은 그대로다.
  - 미소장 아이템은 실루엣·자물쇠로 압박하지 않는다. 꾸미기 화면에는 소장한 것만 보여 주고, 획득 조건 목록을 노출하지 않는다.
  - 알림·홈의 정보 우선순위는 그대로다. 꾸미기는 어디까지나 기록 화면 아래의 선택 기능이다.

### 6.5 저장 스키마 (기기 내)

기존 기록(`teum.records.v1`)과 분리해 `teum.desk.v1`로 저장한다.

```text
version        스키마 버전 (마이그레이션용)
cumulativeDone 누적 챙김 카운터 — 마일스톤 도착 판정의 단일 근거
placements     { slotId: itemId } — 사용자가 바꾼 슬롯만 저장, 없으면 기본 배치
```

소장 여부는 저장하지 않고 `cumulativeDone`과 카탈로그의 `acquire`에서 계산한다 — 저장 상태가 늘수록 마이그레이션 부담만 커진다.

- 서버 전송 없음. `기록 모두 삭제`와는 독립이며, 앱 데이터 삭제 시에만 함께 사라진다.
- 알 수 없는 아이템 id(다운그레이드 등)는 무시하고 기본 배치로 대체한다. 크래시 원인이 되면 안 된다.
- 개인정보처리방침에는 "꾸미기 선택 상태"를 기기 저장 항목으로 한 줄 추가한다 (1.1.x 출시 시).

## 7. HCI·접근성 기준

- 픽셀 장면은 정보를 독점하지 않는 장식/요약 요소로 표시한다.
- 접근성 라벨 예시: `오늘 3번의 틈을 챙겼어요. 책상 장면에 창문 빛이 켜져 있어요.`
- 색만으로 완료 여부를 구분하지 않는다. 슬롯 수와 문장을 함께 제공한다.
- `움직임 줄이기` 설정이 켜지면 모든 프레임 애니메이션을 정지 화면으로 대체한다.
- 장면의 깜빡임 주기는 1초보다 짧지 않으며, 알림 진동과 동기화하지 않는다.
- 한국어 본문은 기존 Pretendard/시스템 글꼴을 유지한다. 픽셀 폰트는 작은 영문 라벨에만 제한한다.

## 8. 구현 단위

### Android

```text
PixelGlyph          도트맵(색상 토큰 2차원 배열) 렌더러
ItemCatalog         아이템 정의 테이블 (3.4 스키마) — 확장 시 여기에만 추가
DeskSlots           슬롯 좌표·허용 크기 정의 (3.3)
PixelScene          슬롯 + 배치된 아이템 인스턴스를 레이어 순서로 합성
PixelSlotMeter      오늘의 8칸
PixelSceneState     doneCount·pause·소장/배치 상태를 장면 상태로 변환
DeskStateStore      teum.desk.v1 로드/저장/마이그레이션 (1.1.x)
```

- React Native `View` 사각형을 사용해 코드로 만든 기본 오브젝트를 먼저 구현한다. **외부 PNG·스프라이트는 사용하지 않는 것을 기본으로 한다** — 브랜드 팔레트 상속, 번들 크기, 라이선스 관리에서 코드 도트맵이 유리하다.
- `PixelGlyph`는 픽셀마다 `View`를 만들지 않는다. **가로로 이어진 같은 색 픽셀을 하나의 사각형으로 합치는(run-length) 방식**으로 그려, 장면 하나가 수십 개의 `View`로 끝나게 한다.
- 예외적으로 외부 자산을 쓸 경우 논리 캔버스에 맞춰 nearest-neighbor로만 확대하고, 필요한 파일만 검토 후 복사한다.
- 사용자의 기록 모드가 꺼져 있으면 `doneCount`는 홈 픽셀 데스크에 사용하지 않는다.

### PC

- CSS `image-rendering: pixelated` 또는 Canvas의 정수 배율 사용
- 메뉴바 팝오버가 열릴 때만 장면 계산
- 운영체제별 애니메이션 감소 설정을 감지하고 정지 장면 제공

## 10. 출시 전 검증 기준

1. 알림 완료 후 추가 보상 화면 없이 2초 안에 홈으로 돌아온다.
2. 기록 모드가 꺼져 있으면 픽셀 데스크와 달력 기록이 노출되지 않는다.
3. `이번엔 넘길게요`를 눌러도 장면이 후퇴하지 않는다.
4. 360dp 폭에서 제목·남은 시간·주요 버튼이 픽셀 장면보다 우선 보인다.
5. TalkBack이 장면을 하나의 요약 문장으로 읽는다.
6. `움직임 줄이기` 설정에서 장면이 깜빡이지 않는다.
7. 원본 오픈소스 전체 디렉터리가 APK/AAB에 들어가지 않는다.
8. `THIRD_PARTY_NOTICES.md`에 실제 복사한 파일만 기록한다.
9. 1.0.4의 오브젝트 4개가 카탈로그·슬롯 구조 위에 구현되어, 새 아이템 추가가 카탈로그 데이터 추가만으로 가능하다.
10. 저장된 배치에 알 수 없는 아이템 id가 있어도 크래시 없이 기본 배치로 대체된다.
