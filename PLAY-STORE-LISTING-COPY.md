# 틈새움 TeuM — Google Play 스토어 카피

> 상태: 한국어 기본 스토어 등록정보 최종 초안  
> 제품 정의: 물 한 모금과 가벼운 스트레칭을 챙기는 직장인 건강 반복 알람  
> 핵심 감정: 건강을 위해 일하는 하루에도 작은 틈이 필요하다.

## 1. 사례에서 가져온 설득 구조

문장을 복사하지 않고, 실제 Play 스토어 건강·알람 앱이 사용하는 설명 순서만 참고했다.

| 사례 | 효과적인 구조 | 틈새움에 적용 | 사용하지 않을 요소 |
| --- | --- | --- | --- |
| [SitBreak](https://play.google.com/store/apps/details?id=com.lifesyncth.sitbreak) | 오래 앉아 있는 현실을 질문으로 환기하고, 자꾸 잊는 문제 뒤에 단순한 해결책을 제시 | `바쁜 날일수록 1분은 저절로 생기지 않는다`는 문제 인식 | 통증·질환 위험 강조, 연속 기록 압박 |
| [Moova](https://play.google.com/store/apps/details?id=app.stretchminder.android) | 바쁜 날에도 웰빙을 우선할 수 있다는 가치 제안 | 업무 중에도 건강을 뒤로 미루지 않는다는 메시지 | 자세 교정·통증 완화·질병 감소 등 강한 건강 효과 주장 |
| [WaterMinder](https://play.google.com/store/apps/details?id=com.funnmedia.waterminder) | 바쁜 일상에서 물을 잊는 문제와 맞춤 알림을 연결 | 물 한 모금을 반복 알람의 구체적인 행동으로 제시 | 섭취량 계산, 신체 수치, 목표 달성·배지 |
| [알라미](https://play.google.com/store/apps/details?hl=ko&id=droom.sleepIfUCan) | 기능보다 원하는 하루의 변화를 먼저 말하고, 이후 섹션별로 기능을 증명 | 건강을 위한 1분이라는 결과를 먼저 말한 뒤 반복 알람 기능 설명 | `무조건`, `절대`, `No.1` 같은 공격적·검증 불가능한 표현 |

공통적으로 좋은 설명은 `사용자의 현실 → 필요한 변화 → 앱의 단순한 해결 → 핵심 기능 → 추천 대상` 순서다. 틈새움은 여기에 죄책감 없는 넘기기와 로컬 개인정보 원칙을 추가한다.

## 2. 카피 체계

| 역할 | 최종 문구 |
| --- | --- |
| 브랜드 태그라인 | `일하는 나를 위한 1분` |
| 감정 카피 | `건강을 위해, 일하는 하루에도 틈은 필요하니까.` |
| 기능 정의 | `물 한 모금과 가벼운 스트레칭을 챙기는 직장인 건강 반복 알람` |
| 알람 화면 | `물 한 모금과 가벼운 스트레칭을 챙길 시간이에요.` |
| 기록 가치 | `챙긴 만큼만, 부담 없이 돌아봐요.` |

`건강`은 필요성을 설명하는 말로 사용한다. `치료`, `예방`, `통증 완화`, `자세 교정`, `집중력 향상 보장`처럼 측정하거나 증명하지 않는 효과는 사용하지 않는다.

## 3. Play Console 입력용 최종안

Google Play 기준은 앱 이름 30자, 짧은 설명 80자, 자세한 설명 4,000자 이하다.

### 앱 이름

```text
틈새움 TeuM
```

### 짧은 설명 — 최종 선택

```text
건강을 위해 필요한 1분, 물 한 모금과 스트레칭을 챙기는 직장인 알람
```

검색어를 억지로 나열하지 않고 `건강`, `물`, `스트레칭`, `직장인`, `알람`을 한 문장 안에서 자연스럽게 설명한다.

### 짧은 설명 — 비교 후보

기능 우선형:

```text
업무 시간마다 물 한 모금과 스트레칭을 챙겨 주는 직장인 반복 알람
```

감정 우선형:

```text
일하는 하루에도 건강한 틈이 필요하니까, 나를 챙기는 1분 알람
```

초기 출시는 필요성과 기능이 함께 드러나는 `최종 선택` 문구를 사용한다. 설치 전환을 비교할 수 있는 충분한 노출이 생긴 뒤에만 후보 문구를 실험한다.

### 자세한 설명 — 최종안

```text
건강은 퇴근 후에만 챙기는 일이 아닙니다.
오래 앉아 일하는 하루 사이에도, 몸을 위한 작은 틈이 필요합니다.

하지만 바쁜 날일수록 그 1분은 저절로 생기지 않습니다.
틈새움은 사용자가 정한 업무 시간에 반복 알람을 울려
물 한 모금과 가벼운 스트레칭을 잊지 않게 알려 주는 직장인 건강 앱입니다.

알람이 울리면 거창한 운동을 시작할 필요는 없습니다.
물 한 모금을 마시거나 자리에서 가볍게 스트레칭해 보세요.
일하는 나를 위한 1분은 그렇게 시작됩니다.

건강한 틈을 놓치지 않도록

- 업무 시작·종료 시간과 반복 요일 설정
- 15분부터 4시간까지 알람 간격 설정
- 다음 건강 알람 시각을 홈에서 바로 확인
- 무음 또는 가볍게·또렷하게·확실하게 진동 선택
- 권한이 허용된 기기에서는 잠금 화면에서도 1분의 틈 표시

알람은 확실하게, 사용은 부담 없이

알람이 울렸을 때 세 가지 중 하나만 고르면 됩니다.

- 5분 뒤에 다시
- 네, 챙겼어요
- 이번엔 넘길게요

매번 완벽하게 해낼 필요는 없습니다.
바쁜 순간에는 미루거나 넘기고, 다시 챙길 수 있을 때 시작하면 됩니다.

이런 분께 잘 맞아요

- 오래 앉아 일하다 보면 물과 스트레칭을 자주 잊는 분
- 정해 둔 업무 시간에만 건강 알람을 받고 싶은 분
- 복잡한 운동 영상보다 짧고 단순한 안내를 원하는 분
- 기록 없이 가볍게 쓰거나, 원할 때만 실천 내역을 보고 싶은 분

기록은 원할 때만

체크와 통계가 번거롭다면 가볍게 사용할 수 있습니다.
기록을 남기고 싶다면 O/X 결과를 기기에 저장하고,
달력에서 날짜별 횟수와 시간대별 내역을 확인할 수 있습니다.

개인정보는 기기 안에

- 계정과 회원가입이 없습니다.
- 광고·분석을 위한 사용 기록을 서버로 보내지 않습니다.
- 설정과 선택형 O/X 기록은 사용자의 기기에만 저장됩니다.
- 기록은 설정에서 모두 삭제할 수 있습니다.

틈새움은 의료 서비스나 진단 도구가 아니며,
건강 상태나 물 섭취량을 측정하지 않습니다.

건강을 위해, 일하는 하루에도 틈은 필요하니까.
일하는 나를 위한 1분, 틈새움 TeuM.
```

## 4. 스크린샷 카피

한 화면에는 제목 한 줄과 보조 문장 한 줄만 사용한다. 실제 앱 화면을 가리지 않으며, 스크린샷끼리 같은 말을 반복하지 않는다.

### 1장 — 건강 필요성

제목:

```text
일하는 하루에도 건강한 틈이 필요하니까
```

보조 문장:

```text
물 한 모금과 가벼운 스트레칭, 1분부터 챙겨요
```

### 2장 — 반복 알람

제목:

```text
내 업무에 맞춘 건강 알람
```

보조 문장:

```text
업무 시간과 요일, 알람 간격을 직접 정해요
```

### 3장 — 예측 가능성

제목:

```text
다음 건강 알람을 한눈에
```

보조 문장:

```text
언제 울릴지 알고, 필요할 때 잠시 멈출 수 있어요
```

### 4장 — 1분의 틈

제목:

```text
알람이 울리면, 1분의 틈
```

보조 문장:

```text
지금 챙기거나, 5분 뒤로 미루거나, 가볍게 넘겨요
```

### 5장 — 선택형 기록

제목:

```text
챙긴 만큼만, 부담 없이 돌아봐요
```

보조 문장:

```text
원할 때만 기록하고 날짜와 시간별로 확인해요
```

## 5. Feature graphic과 광고용 문구

Feature graphic의 브랜드 문구는 기존대로 유지한다.

```text
일하는 나를 위한 1분
```

다른 광고 소재에서 사용할 보조 카피:

```text
건강을 위해, 일하는 하루에도 틈은 필요하니까.
```

```text
바쁜 날일수록 몸을 위한 1분은 저절로 생기지 않으니까.
```

```text
물 한 모금, 가벼운 스트레칭. 잊지 않게 울리는 건강 알람.
```

한 소재에 위 문구를 두 개 이상 함께 쓰지 않는다.

## 6. Play 등록 전 카피 검수

- 앱에 아직 없는 기능을 설명하지 않는다.
- `매시간`을 사용할 때는 기본 간격이 60분이며 사용자가 바꿀 수 있음을 상세 설명에서 밝힌다.
- 전체 화면 표시는 기기와 사용자 권한에 따라 달라지므로 `항상`, `무조건`이라고 쓰지 않는다.
- 익명 후기, 다운로드 수, 순위, 수상 경력을 만들지 않는다.
- 질병·통증·자세 교정·집중력 개선을 보장하지 않는다.
- 짧은 설명에 이모지, 반복 기호, 설치 요청, 과도한 검색어를 넣지 않는다.
- 실제 출시 빌드에 O/X·달력이 없다면 해당 문단과 5번 스크린샷을 제거한다.
